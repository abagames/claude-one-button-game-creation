# GRAVITY DRIFT (gravity-drift)

**タグ**: #field-regenerate, #on_got_item-reverse_state, #field-space

## 1. コア・メカニクス

**入力**: ボタン押下で推力発生（上方向）
**挙動**: 宇宙空間で慣性移動。重力は弱く、推力で位置を調整
**終了条件**: 隕石（障害物）に衝突
**スコアリング**: 生存時間 + 星（アイテム）収集
**難度上昇**: 隕石の出現頻度と速度が増加

### 特殊メカニクス
- **重力反転**: 星を取ると一定時間、隕石が逆方向に流れる
- **再生する隕石**: 画面外に出た隕石は再生成され続ける

## 2. オブジェクト仕様

| オブジェクト | 形状 | 動作 | 衝突処理 |
|:---|:---|:---|:---|
| Player | 小さな三角形（宇宙船） | 慣性移動、ボタンで上推力 | 隕石→ゲームオーバー |
| Asteroid | 四角形（赤） | 右から左へ流れる、反転時は左から右 | プレイヤー→終了 |
| Star | 小さな黄色box | ゆっくり流れる | プレイヤー→スコア+重力反転 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| シンプルさと直感性 | ◎ 1ボタンで推力、慣性で漂流 |
| 視覚的フィードバック | ◎ 反転時に隕石の流れが逆転（明確） |
| スキルベースのスコアリング | ○ 星を取るリスクと反転ボーナスのトレードオフ |
| 斬新なメカニクス | ◎ 障害物の流れ方向が反転する新規性 |

## 4. タグとの関係

- **field-space**: 慣性が強い宇宙空間での漂流感
- **on_got_item-reverse_state**: 星取得で隕石の流れが反転
- **field-regenerate**: 隕石は無限に再生成される

## 5. 新規性の根拠

従来の避けゲーは「障害物の動きは固定」だが、本作はアイテム取得で障害物の流れ方向自体が反転する。プレイヤーは「今どちら向きに避けるべきか」を常に判断する必要があり、単純な反射神経ではなく状況認識が求められる。
