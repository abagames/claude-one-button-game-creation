# TILTED (tilted)

**タグ**: #on_pressed-jump, #on_pressed-turn, #field-floors, #field-bottomless, #rule-physics, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップでジャンプ、空中2回目以降で反転＋ジャンプ、長押しで重力軽減 |
| 挙動 | 傾いたバーを渡りながら上昇、ボール回避、空中反転で方向転換、底なし落下で死亡 |
| 終了条件 | 画面下端到達 |
| スコア | 生存時間と上昇距離に応じて自動加算（scr = 0.02d + (60-y)*0.1） |
| 難易度 | 時間経過でスクロール速度、ジャンプ速度、ボール出現頻度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("a","b","c") 黒 | バー上：左右移動、空中：放物線＋反転ジャンプ、重力0.1（長押し0.05） | - |
| bar | bar(length, 3, angle) 薄黒 | 上から降下、傾き±π/5、中央or左右対称配置 | プレイヤー着地、ボール反射 |
| ball | char("d") 黄 | 上から降下、物理挙動（重力0.05、減衰0.995）、画面端反射 | プレイヤー：押し下げ、ボール同士：合体（char "e"） |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 傾いたバーの視覚的インパクト、ジャンプと反転の明確な動作、ボールの物理的動き、底なし感の恐怖 |
| (2) 視覚的フィードバック | バー傾き角度、空中時の動作変化、ボール衝突の押し下げ、ボール合体、画面下端の危険ライン |
| (3) リスク・リワード | 低位置滞在で高スコア vs 高位置で余裕、ボール近接でバー渡り vs 安全待機、反転ジャンプで精密移動 |
| (4) 斬新なメカニクス | 空中反転システム（2回目以降で方向転換＋減衰ジャンプ）、ボール合体メカニクス、位置依存スクロール加速 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #on_pressed-jump | バー上：ジャンプ（vy = -2）、空中1回目：2段ジャンプ（vy = -2）、長押しで重力0.05 vs 0.1 | ジャンプタイミングで軌道調整、長押し滞空時間延長、バー間移動の精密制御 |
| #on_pressed-turn | 空中2回目以降タップで左右反転（vx *= -1）＋弱ジャンプ（vy = -2/cnt^2） | 空中方向転換、減衰で連続反転防止、計画的な軌道修正 vs 緊急回避 |
| #field-floors | バーが上から降下、傾き0～±π/5、長さ15-22、中央単体or左右対称ペア配置 | 傾きでプレイヤー滑り、左右非対称な配置、予測と反応の混合要求 |
| #field-bottomless | 画面下端（y > 99）で即死、上方向のみ安全 | 常に上昇圧力、スクロール速度競争、下降リスクを恐怖で強調 |
| #rule-physics | ボール・プレイヤーに重力・慣性・反射、バーで反射、ボール合体（接触でchar "e"） | 物理予測の習熟曲線、ボール軌道読み、合体で危険増加 |
| #field-auto_scroll | 上スクロール（scr = 0.02d + (60-y)*0.1）、プレイヤーが低いほど加速 | 低位置ハイリスク、高位置余裕、位置取り戦略、得点とリスクのトレードオフ |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ボールを足場に変換、タップで破壊して一時的なバーに、資源管理と軌道予測 |
| Combine (結合) | バーに色属性追加、プレイヤーの色と一致で摩擦増加、不一致で滑る、色選択システム |
| Modify (修正) | 反転ジャンプ回数制限（3回まで）、バー着地でリチャージ、計画的な空中移動要求 |
