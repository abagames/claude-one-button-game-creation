# DANGO (dango)

**タグ**: rule-mathematics, on_pressed-pierce, rule-time_limit

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップで串を伸ばす |
| 挙動 | タップで串を伸ばし、触れた数式団子を串刺しにして計算式を組み立てる。串は自動で引っ込む |
| 終了条件 | 残り串数が0未満 |
| スコア | 完成した数式の計算結果。100点ごとに串+1 |
| 難易度 | 団子の移動速度上昇、伸縮速度上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| stick (青) | rect(伸縮) | タップで右から左へ伸縮。10フレーム伸び、10フレーム縮む | - |
| deadline (水色) | rect | 固定位置 | 団子接触でゲームオーバー |
| dango | box + text | 画面左から出現、揺れながら右へ移動 | 串刺しで式に追加 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 団子に書かれた数式(+3, *5など)が明確。串を伸ばして刺すという物理的な直感性 |
| (2) 視覚的フィードバック | 画面下部に組み立て中の式、右下に計算結果を表示。団子が締切線に近づく緊張感が視覚的 |
| (3) リスク・リワード | 大きな数を狙うか安全な小さな数か。*演算子は高得点だが先頭に来ると0倍になるリスク |
| (4) 斬新なメカニクス | 数式を物理的に「串刺し」で組み立てる独特の数学パズル。時間制限と計算の最適化の融合 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| rule-mathematics | 数式を組み立てて計算 | 単なる反射神経だけでなく、数式の評価と最適化を要求。算数の理解がスコアに直結 |
| on_pressed-pierce | タップで串を伸ばして団子を貫通 | タイミングと位置の正確性。複数の団子を一度に刺す戦略も可能 |
| rule-time_limit | 団子が締切線に到達するまでの制限 | 熟考と迅速な判断のバランス。100点ごとの串追加で時間との戦いが続く |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 数式を「色の組み合わせ」に置き換え、RGB値を調整して目標色を作る。串で色団子を刺して混色 |
| Combine (結合) | rule:combo_multiplierを追加し、連続して団子を刺すと倍率上昇。ミスで倍率リセット |
| Modify (修正) | 括弧演算子()を追加し、優先順位を制御可能に。より複雑な数式の構築と高得点の可能性 |
