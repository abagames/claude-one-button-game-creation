# EAROCK (earock)

**タグ**: field-outpost, field-gravity, on_holding-thrust, field-space

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップで推力噴射 / 離すと惑星に自動旋回 |
| 挙動 | タップで推力方向へ加速。離すと中央の惑星に向かって自動旋回 |
| 終了条件 | 星が惑星に衝突 |
| スコア | 星を取得時に加算スコア(+1, +2, +3...)を獲得。壁/惑星反射でリセット |
| 難易度 | 水平速度増加、星の出現頻度・速度増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| ship | bar×2 + bar | 慣性飛行。タップで推力、離すと惑星へ旋回 | - |
| planet (中央) | arc(blue/green) | 固定。重力中心 | 反射、加算スコアリセット |
| stars | char(a) | 惑星周辺から出現、重力で軌道変化 | +加算スコア点 |
| wall | 画面端 | - | 反射、加算スコアリセット |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 中央惑星、宇宙船、星のみ。タップで推力、離すと自動旋回という明快な2モード |
| (2) 視覚的フィードバック | パーティクルで推力方向を表示。左下に次の加算スコア(+1, +2...)を表示し、目標を明確化 |
| (3) リスク・リワード | 連続取得で加算スコア上昇だが、反射でリセット。惑星や壁を避けつつ星を追う高度な軌道制御 |
| (4) 斬新なメカニクス | 「離すと自動旋回」による半自動操縦。プレイヤーは推力のタイミングのみ制御し、旋回はシステムに委ねる |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| field-outpost | 中央の惑星が重力源かつ障害物 | 惑星は守るべき基地ではなく、引力を利用しつつ衝突を避ける対象。軌道力学の理解が重要 |
| field-gravity | 惑星と星に重力効果 | リアルな軌道運動。重力を利用した軌道設計とタイミング制御 |
| on_holding-thrust | タップで推力発生 | 連続的な加速ではなく、瞬間的な推力による軌道修正。燃料の概念はなく純粋なタイミングゲーム |
| field-space | 宇宙空間での慣性飛行 | 摩擦のない環境での運動。一度加速したら減速しないため、慎重な操作が必要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 惑星を「ブラックホール」に置き換え、一定距離内で急激に引き込まれる。星はブラックホールから放出される |
| Combine (結合) | field:rotateを追加し、惑星自体が回転。惑星表面の安全地帯と危険地帯を動的に変化させる |
| Modify (修正) | 複数の小惑星を追加し、それぞれが弱い重力を持つ。複雑な多体問題による予測困難な軌道 |
