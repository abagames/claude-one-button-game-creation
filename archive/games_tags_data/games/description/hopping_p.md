# HOPPING P (hopping_p)

**タグ**: player-bounce, obstacle-bounce, on_got_item-power_up

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | タップで左右移動方向を反転、地面着地時に自動ホッピング（跳ねる） |
| 終了条件 | パワーアップ無効時に敵と接触 |
| スコア | パワーアップ中に敵を倒すと得点、倍率は2倍ずつ上昇（最大64倍） |
| 難易度 | 時間経過で敵の出現頻度と移動速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | 文字"o"（緑） | 左右ホッピング、タップで方向反転、パワーアップ中は倍速 | - |
| enemy | 文字"x"（赤） | 左右ホッピング、4種類の跳躍パターン | 通常時は終了、パワーアップ中は+得点 |
| power | 文字"P"（黄） | 左右ホッピング | パワーアップ発動、全敵の方向反転 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | テキスト文字とシンプルな色分け、ホッピング動作の視認性が高い |
| (2) 視覚的フィードバック | パワーアップ中は敵が青に変色、音とパーティクルで状態変化を強調 |
| (3) リスク・リワード | パワーアップ中の敵討伐で倍率上昇、但し制限時間内に多く倒す必要 |
| (4) 斬新なメカニクス | 複数キャラクターが同時にホッピング、軌道予測と衝突回避の判断 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| player-bounce | 地面着地時に自動ホッピング | 跳躍のタイミングを予測し、空中で方向転換の判断が必要 |
| obstacle-bounce | 敵も同様にホッピング移動 | 敵の跳躍パターン（4種類）を見極め、衝突を回避する軌道選択 |
| on_got_item-power_up | パワーアップ取得で状態反転 | パワーアップの取得タイミングと、制限時間内の効率的な敵討伐 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ホッピングを円弧軌道に変え、円形フィールドで回転移動 |
| Combine (結合) | 敵の色を複数種類に分け、同色の敵のみ倒せるマッチングルール |
| Modify (修正) | タップで重力方向を反転させ、天井と床の両方でホッピング |
