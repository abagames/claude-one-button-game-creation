# D LASER (d_laser)

**タグ**: rule-shelter, rule-gauge_management, on_pressed-turn, on_holding-stop, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（方向転換） / 長押し（停止） |
| 挙動 | プレイヤーは画面下部を左右に移動し、画面上部から発射される赤いレーザーを避ける。青いシールドが出現し、レーザーを遮断する。 |
| 終了条件 | レーザーに接触、または画面下部に押し出される |
| スコア | レーザー発射中に移動した距離に応じて得点 |
| 難易度 | 時間経過とともにレーザーチャージ速度と移動速度が上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("a"/"b", 6x6) | 左右に自動移動。押すと方向転換、長押しで停止。押していないときは上方向にも移動 | - |
| laser | rect(10幅) | 画面上部から垂直に発射。シールドに当たると停止して遅い速度で移動 | 接触で終了 |
| shield | box(8,4) + char("c") | 画面上部から出現し、ランダムな横位置に移動してレーザーを防ぐ | レーザーを遮断 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | レーザーは赤、シールドは青と色分けが明確。画面上部のゲージでレーザー発射タイミングを視覚化 |
| (2) 視覚的フィードバック | レーザーチャージゲージ（赤いバー）が溜まると発射。シールドが停止位置に到達すると濃い青に変化 |
| (3) リスク・リワード | ボタンを離して移動することで距離を稼ぎスコアを得るが、レーザー回避が難しくなる。シールドの配置を見極める判断力が重要 |
| (4) 斬新なメカニクス | シールドによる「避難所」の概念と、ボタン長押しで停止する逆転的な操作。移動とスコアの直結によるリスク管理 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| rule-shelter | シールドがレーザーを遮断し安全地帯を作る | シールドの配置を予測し、適切なタイミングで移動する判断力が求められる |
| rule-gauge_management | 画面上部のレーザーチャージゲージを監視 | ゲージの動きを読み、レーザー発射前にシールド下に移動する計画性が重要 |
| on_pressed-turn | ボタンを押すと移動方向を反転 | 画面端での反転や、急な方向転換でレーザーを回避する瞬発力が試される |
| on_holding-stop | ボタンを長押しすると停止 | シールド下で安全に待機したり、レーザーの隙間で微調整するための重要な操作 |
| field-auto_scroll | 画面が上方向にスクロールし、プレイヤーが押し下げられる | ボタンを離して上昇することでスクロールに抗する必要があり、積極的な移動を促す |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | レーザーをミサイルやビームに、シールドをバリア発生装置やトンネルに置き換える |
| Combine (結合) | 複数のゲージを管理（レーザー、シールド生成、スコアボーナスタイムなど）してより複雑な判断を要求 |
| Modify (修正) | シールドが移動中のみ有効にする、レーザーが複数同時発射される、プレイヤーが上下移動もできるようにする |
