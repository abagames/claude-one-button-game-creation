# CIRCLE W (circle_w)

**タグ**: player-circle, field-pins, obstacle-penalty, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | 円形プレイヤーの半径を変更（長押しで拡大+0.5、離すと縮小-0.5）し、他の円に接触して乗り移る |
| 終了条件 | 赤い円（危険）に接触 |
| スコア | 青い円に接触すると倍率分の得点、接触時の半径（ceil）が倍率に加算 |
| 難易度 | 時間経過でスクロール速度増加（√difficulty） |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | arc(radius) 緑 | 半径可変（1-9以上）、位置に応じて画面スクロール | - |
| circle (normal) | arc(8-15) 青 | 自動スクロール、縦方向にラップアラウンド | プレイヤー化、+multiplier点 |
| circle (danger) | arc(8-15) 赤 | 24-30個ごとに1個出現、自動スクロール | ゲームオーバー |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 円のサイズ変更という直感的な操作、色分け（青=安全、赤=危険、緑=プレイヤー） |
| (2) 視覚的フィードバック | 半径のリアルタイム変化、倍率表示、円の接触判定の視覚的明確さ |
| (3) リスク・リワード | 大きな半径で接触すると高倍率だが接触リスク増、小さな半径で安全だが倍率減少 |
| (4) 斬新なメカニクス | 円から円へ乗り移る、半径変更による当たり判定の動的制御、半径減少による倍率ペナルティ |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| player-circle | 円形の当たり判定、半径可変（0.5×√difficulty/フレーム） | サイズ調整による当たり判定制御、大小のトレードオフ、動的な難易度調整 |
| field-pins | 円（ピン）が複数配置、ランダム位置（y方向）、半径8-15のバリエーション | 配置パターンの読み、衝突回避の戦略、安全ルートの選択 |
| obstacle-penalty | 半径9未満で倍率減少、赤い円に接触でゲームオーバー | 倍率維持のプレッシャー、赤い円の回避戦略、リスク管理 |
| field-auto_scroll | 画面が自動的に左にスクロール（-√difficulty）、縦方向にラップアラウンド | 時間的プレッシャー、画面外に出る前の乗り移り、縦方向の自由度 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 円を四角や三角に変更 / 半径変更を色変更に変更して異なる色の円に乗り移る |
| Combine (結合) | 複数のプレイヤー円を同時操作 / 円の種類を増やして特殊効果を追加 |
| Modify (修正) | 半径の最大値を制限してリスク管理を強化 / 倍率ペナルティの閾値を変更 / 赤い円の出現頻度を調整 |
