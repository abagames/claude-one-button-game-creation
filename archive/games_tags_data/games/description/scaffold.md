# SCAFFOLD (scaffold)

**タグ**: #player-scaffold, #on_pressed-turn, #on_holding-extend, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップで足場の角度変更 / 長押しで足場設置 |
| 挙動 | プレイヤーは足場の上を歩く。タップで次の足場の角度（下り/平坦/上り）を変更し、長押しで足場を設置。足場を設置し続けることで前進。壁の隙間を通過し、コインを取得してスコアを稼ぐ。 |
| 終了条件 | 壁またはトゲに接触 |
| スコア | コインを取得で得点（マルチプライヤーあり） |
| 難易度 | 時間経過でスクロール速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | カスタムキャラ（d/e） | 足場の上を歩く、自動的に左へ戻ろうとする | - |
| floors | カスタムキャラ（a/b/c） | 長押しで設置、3種類（下り/平坦/上り） | プレイヤーを支える |
| wall | rect(赤) | 隙間を持つ壁、左からスクロール | 接触で終了 |
| objs | text($/x) | コイン（黄）またはトゲ（赤）、上下に往復移動 | コイン: 得点 / トゲ: 終了 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 足場のキャラクターが3種類（上り/平坦/下り）で視覚的に明確。壁の隙間、コイン（$）、トゲ（x）もシンプルで理解しやすい。 |
| (2) 視覚的フィードバック | 足場を設置する音、コイン取得の音、ゲームオーバーの音が明確。足場の角度変更時の色の変化（cyan）で次の足場が予告される。 |
| (3) リスク・リワード | コインを取るために足場を適切に配置する必要があり、壁の隙間を通過するためには足場の高さを調整する必要がある。コインを取り逃すとマルチプライヤーが減少するペナルティ。 |
| (4) 斬新なメカニクス | 足場を設置しながら進むという独特のメカニクス。足場の角度を選択して高低差を作りながら進む。自分で道を作るという創造的な要素が面白い。 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-scaffold | プレイヤーは足場を設置しながら進む、足場がなければ落下 | 足場の配置計画が重要。先を見越して適切な高さに足場を設置する必要がある |
| #on_pressed-turn | タップで足場の角度を変更（下り→平坦→上り→平坦と巡回） | 壁の隙間やコインの位置を見て、適切な角度を選択する判断力が求められる |
| #on_holding-extend | 長押しで足場を連続設置、一定時間ごとに1つ設置 | 長押しの時間を調整して、必要な分だけ足場を設置する。連打ではなく長押しという点が独特 |
| #field-auto_scroll | 画面が自動スクロール、足場も左へ移動 | スクロールに追いつくように足場を設置し続ける必要があり、リズム感が求められる |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 足場の角度を別の要素（色や形状の違いで異なる効果）に置き換え |
| Combine (結合) | 足場の設置以外のアクション（ジャンプやダッシュ）を追加し、より多様な移動を可能にする |
| Modify (修正) | 足場の種類を増やして5-6種類にし、より複雑な高低差を作れるようにする |
