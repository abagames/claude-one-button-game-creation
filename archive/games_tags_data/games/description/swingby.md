# SWINGBY (swingby)

**タグ**: #field-gravity, #on_holding-rotate, #field-space

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | 宇宙空間を飛行する宇宙船を操作。星の重力に引かれながら、ボタン長押しで右方向に旋回。常に前方への推進力が働く |
| 終了条件 | 星に連続衝突してダメージ蓄積値が99を超える |
| スコア | 原点からの直線距離がリアルタイムにスコアとして表示 |
| 難易度 | 時間経過で重力の強さ、推進力、衝突ダメージが増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| ship | bar(3,3) 灰色 | 常に前方へ推進、ボタン長押しで右旋回、星の重力に引かれる | ダメージ蓄積+4×difficulty |
| star | arc(半径5-15) 黒 | 宇宙空間に配置、重力場を発生 | 上記参照 |
| home | box(10,10) 灰色 | 原点（スタート地点）、画面内に表示 | - |
| bg_star | rect(1,1) 灰黒 | 背景の星、視差効果で船の移動に応じて動く | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 重力という物理法則を利用した直感的なゲームプレイ。ボタン長押しで右旋回というシンプルな操作。背景の視差効果で速度感を表現 |
| (2) 視覚的フィードバック | 旋回時と推進時のパーティクル、衝突時のパーティクル量でダメージを表現。カメラが速度に応じてオフセットし、スピード感を強調 |
| (3) リスク・リワード | 星に近づくほど重力で大きく加速できるが衝突のリスクも増大。スイングバイ軌道を使って効率的に遠くまで飛ぶ技術が高得点に繋がる |
| (4) 斬新なメカニクス | 実際の宇宙物理のスイングバイを模した重力アシスト。右旋回のみという制約が、軌道計画に戦略性をもたらす |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-gravity | 各星が重力場を発生し、宇宙船を引き寄せる | 重力を利用したスイングバイ軌道で効率的に加速。星の配置を先読みし、最適な軌道を計画する必要がある |
| #on_holding-rotate | ボタン長押しで進行方向に対して右方向に旋回 | 左旋回ができない制約により、軌道修正のタイミングが重要。連続的な右旋回で大きく方向転換する技術が必要 |
| #field-space | 無限の宇宙空間、境界なし | 慣性が維持されるため、一度加速すると止まらない。長期的な軌道予測と、原点からの距離という明確な目標が戦略を生む |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 星をブラックホール（強重力）と惑星（弱重力）に分け、色で区別する |
| Combine (結合) | チェックポイント（小惑星）を配置し、通過することでボーナス点を獲得する探索要素を追加 |
| Modify (修正) | 左右両方向に旋回可能にする代わりに、燃料制限を設けてリソース管理要素を加える |
