# SUM TEN (sum_ten)

**タグ**: #rule-mathematics, #on_pressed-go_forward

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | 数字が配置されたグリッド上をカーソルが自動移動。タップで次のマスに前進し数字を取得。取得した数字の合計が10の倍数になるよう調整 |
| 終了条件 | カーソルが画面端に到達、または10個の数字を取得しても合計が10の倍数にならない |
| スコア | 合計が10の倍数になると (合計÷10)² × 10 点を獲得 |
| 難易度 | 時間経過でグリッドの左上方向へのスクロール速度が上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| cursor | rect(11,11) 水色 | グリッド上を自動で左または上方向に移動、30tick毎に方向転換 | - |
| next | rect(11,11) 赤色 | カーソルの次の移動先を示す | タップで前進、数字取得 |
| number | text(1-9) 青色 | グリッド上にランダム配置、取得されると穴になる | 数字を取得リストに追加 |
| hole | rect(11,11) 白塗り | 数字がない空きマス、カーソルは通過不可 | カーソルは方向転換 |
| grid | rect可変 | 左上方向に自動スクロール、新しい行・列が右下から追加 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 数字を足して10の倍数にするという明快なルール。カーソル（水色）と次の位置（赤）の色分けで行動が予測可能。画面右側に現在の合計を常に表示 |
| (2) 視覚的フィードバック | 10の倍数達成時の効果音とスコア表示。取得リストが画面右に積み上がり、進捗が視覚化される。合計が60tick表示された後にリセット |
| (3) リスク・リワード | 大きな数を集めて高い合計（30, 40等）を狙うとボーナス点が大きいが、10の倍数を逃すリスクも増大。少ない数で確実に10を作るか、大胆に攻めるかの判断 |
| (4) 斬新なメカニクス | 自動移動するカーソルと、左または上という制限された移動方向。スクロールにより時間制限と空間制約が同時に働く独特のプレッシャー |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #rule-mathematics | 取得した数字の合計を10の倍数にする算術パズル | 先読みして目標の数を逆算する計算能力と、現在の合計を常に把握する記憶力が求められる。数学的思考がスコアに直結 |
| #on_pressed-go_forward | タップで自動移動しているカーソルを次のマスに進める | 移動タイミングの判断が重要。進まないと画面端に追い詰められ、進みすぎると不要な数字を取得してしまう |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 10の倍数ではなく、素数を作る数字パズルにする。2, 3, 5, 7, 11 などを目標値に |
| Combine (結合) | 数字に加えて演算子（+, -, ×）を配置し、より複雑な式を作るパズル要素を追加 |
| Modify (修正) | プレイヤーが移動方向を選択できるようにし、より戦略的なルート選択を可能にする |
