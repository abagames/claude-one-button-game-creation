# UD CAVE (ud_cave)

**タグ**: #field-multiple, #player-inverted, #on_holding-reverse_state

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し：右に移動 / 離す：左に移動 |
| 挙動 | 上下2つの洞窟を同時に操作。下の洞窟は左右反転した挙動。長押しで右、離すと左に移動し、壁を避けながらゴールドを集める |
| 終了条件 | 上または下の洞窟で壁に衝突 |
| スコア | 壁セグメントを通過する度にマルチプライヤー分のスコア獲得 |
| 難易度 | 時間経過で移動速度とスクロール速度が上昇し、洞窟の幅が狭くなる |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| players(a) | char("a") 黒色 | 上下2つ同時に存在。長押しで右、離すと左に移動。下側は左右反転した挙動 | - |
| walls | rect 赤色 | 上下から不規則にスクロール。洞窟の幅と位置がランダムに変動 | 終了 |
| gold($) | char("$") 黄色 | 洞窟内にランダム出現。上または下のどちらかに出現 | マルチプライヤー+1 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 上下2つの洞窟、黒いプレイヤー、赤い壁、黄色のゴールドという明確な色分け。長押しで右、離すと左という二択の操作 |
| (2) 視覚的フィードバック | 長押しで「select」音、離すと「laser」音。ゴールド取得で「powerUp」音。壁衝突で「explosion」音。マルチプライヤー表示で成長が可視化 |
| (3) リスク・リワード | ゴールドを取るとマルチプライヤー増加で高スコアだが、軌道を変える必要がありリスク増大 |
| (4) 斬新なメカニクス | 上下同時操作かつ下側が左右反転。片方の操作がもう片方に逆の影響を与える独特の認知負荷 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-multiple | 上下2つの洞窟エリアを同時に操作 | 両方のエリアの状況を同時に把握し、どちらも安全な操作を選ぶ判断力が必要 |
| #player-inverted | 下側のプレイヤーが左右反転した挙動 | 反転操作への適応が攻略の核心。上下の動きを同時に予測する認知的チャレンジ |
| #on_holding-reverse_state | 長押しと離すで移動方向が反転 | 押しっぱなしではなく、押す/離すのリズムが重要。タイミング調整が生存とスコアを左右 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 上下を左右に変更。または3つ、4つのエリアに増やし、複数の反転操作を同時に要求 |
| Combine (結合) | 敵や障害物を追加。壁だけでなく、移動する敵を避ける要素を加えて複雑化 |
| Modify (修正) | 反転の仕方を変更。角度反転、速度反転など、異なる種類の反転操作で新しい挑戦を提供 |
