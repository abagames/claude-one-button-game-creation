# BOARDING (boarding)

**タグ**: rule-classify, rule-physics, field-pins, on_holding-stop, on_pressed-reverse_state, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | ホールドで乗客投下・速度低下 / リリースで色切り替え |
| 挙動 | 左右に移動する発射台から赤/青の乗客を投下。ボードで跳ね返し、対応する色のゲートに分類着地 |
| 終了条件 | ゲートが画面上端(y=12)に到達 |
| スコア | 乗客を対応する色のゲートに着地させると得点。ボードでの跳ね返り回数(bc)がスコア。間違った色だと-(1+bc)点、ゲートが上昇 |
| 難易度 | 時間経過でゲートが上昇(0.1×difficulty)。正解でゲート下降、ボードも増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("ab/cd") | 左右往復移動、色切り替え | - |
| passengers | char("ab/cd") | 物理演算で落下 | ボードで反射 |
| boards | char("ef"斜め) | 固定位置、上にスクロール | 乗客を斜めに反射 |
| gates | rect(赤/青) | 下部、上下移動 | 正解で得点・下降 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 赤/青の色分けで分類先が明確。斜めボードの角度で反射方向が予測可能 |
| (2) 視覚的フィードバック | ゲートの上下移動で危機感。スコア表示位置で正解/不正解が明確 |
| (3) リスク・リワード | ボードで多く跳ね返すほど高得点だが、制御困難に。間違うとペナルティも増大 |
| (4) 斬新なメカニクス | 物理パズルと分類ゲームの融合。ボードの配置運と技術の両立 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| rule-classify | 乗客を色で分類 | 正しい判断と色の切り替えタイミングが重要 |
| rule-physics | 物理演算による落下と反射 | 軌道予測と反射角度の理解が必要 |
| field-pins | 固定された斜めボード | ボードの配置を記憶し、軌道を計算する戦略性 |
| on_holding-stop | ホールドで移動速度低下 | 正確な投下位置の調整が可能 |
| on_pressed-reverse_state | リリースで色切り替え | 連続投下と色切り替えのタイミング管理 |
| field-auto_scroll | ゲートとボードが上昇 | 時間制限の圧力。ミスすると加速 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ボードを回転させ、角度が動的に変化 |
| Combine (結合) | 3色以上の分類を追加し、複雑な判断を要求 |
| Modify (修正) | ボードの角度を自分で配置できる設置フェーズを追加 |
