# PAKU PAKU (paku_paku)

**タグ**: #field-1D, #on_got_item-power_up, #obstacle-chase, #on_pressed-turn

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（左右に方向転換） |
| 挙動 | 1次元のフィールド（横線）を左右に移動し、16個のドットを食べる。パワードット（大きい黄色）を食べると約120フレーム間、敵を食べられる |
| 終了条件 | 通常状態で敵に接触 |
| スコア | ドットを食べるとmultiplier点、パワーアップ状態で敵を食べると10×multiplier点、全16個のドットを取得するとmultiplier+1 |
| 難易度 | difficultyに応じて移動速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char(a-c, green) | 左右に移動、画面端でループ、アニメーション4フレーム | - |
| enemy | char(d-e/h) | 赤(通常)/青(パワーアップ)/黒(目)、プレイヤーを追跡または逃走、画面端で停止 | 通常時は終了、パワーアップ時は10×multiplier点獲得 |
| dots | char(f, yellow) | 16個、固定位置、取得でmultiplier点 | 取得で消滅 |
| power_dots | char(g, yellow, 点滅) | 16個中1個、取得で120フレームのパワーアップ | 取得で消滅 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | パックマンの1D版、誰でも知っているルールを1次元に圧縮。色分け（プレイヤー緑、敵赤/青、ドット黄）で役割明確 |
| (2) 視覚的フィードバック | パワーアップ時の敵の色変化（赤→青）、効果終了前の点滅警告、効果音で状態変化を通知 |
| (3) リスク・リワード | パワードットを取ると敵を食べられるが、タイミングを誤ると逆に追い詰められる。全ドット取得でmultiplier増加 |
| (4) 斬新なメカニクス | 3Dのパックマンを1Dに圧縮した独特の視点。左右の移動だけで追跡と逃走の駆け引きが成立 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-1D | 横1本の線上を左右に移動、画面端でループ | 移動方向が限定されるため、敵との位置関係を常に意識する必要がある |
| #on_got_item-power_up | パワードットを取得すると約120フレーム間、敵を食べられる | パワーアップのタイミングが戦略の核。早すぎても遅すぎても敵を食べ損ねる |
| #obstacle-chase | 敵がプレイヤーを追跡（通常時）または逃走（パワーアップ時） | 追跡パターンを読んで、パワードットの位置と敵の位置を計算する必要がある |
| #on_pressed-turn | ボタンを押すと左右に方向転換 | 1Dフィールドでは方向転換のタイミングが生死を分ける。敵との距離を見極めて即座に判断 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 敵を複数体に増やし、それぞれ異なる追跡パターンを持たせる |
| Combine (結合) | 2Dフィールドに拡張し、複数の横線を上下に移動可能にする |
| Modify (修正) | パワーアップ時間を残りドット数に応じて変化させ、戦略的な取得順を要求する |
