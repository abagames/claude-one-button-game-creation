# R WHEEL (r_wheel)

**タグ**: #weapon-change_field, #weapon-vertical, #field-spike, #field-rotate, #on_pressed-jump, #on_pressed-shoot, #on_got_item-power_up

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップでジャンプ、長押しで重力調整 |
| 挙動 | 回転する車輪の外側を歩き、ジャンプで紫バーを生成してスパイクを育て、ボーナスで黄バーを生成してスパイクを破壊 |
| 終了条件 | プレイヤーがスパイクに衝突 |
| スコア | スパイクを破壊すると、スパイクの高さ × multiplier（連続破壊で増加） |
| 難易度 | 車輪の回転速度とジャンプ力が上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("a", "b") | 車輪の外側を歩く、ジャンプで上昇 | スパイクで終了 |
| wheel | arc(40, 3) | 一定速度で回転 | - |
| spike | line（赤） | 紫バーが当たると成長（最大高さ不明） | 黄バーで破壊→得点 |
| bonus | char("c") | 車輪内を回転 | 取得→黄バー生成 |
| bar | box(width, 3) | 下方向に落下、紫は遅く黄は速い | 紫→スパイク成長、黄→スパイク破壊 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 回転する車輪、赤いスパイク、黄色のボーナス、紫と黄のバーと色分けが明確。ジャンプでバーを生成する仕組みが直感的。 |
| (2) 視覚的フィードバック | スパイクが段階的に成長し、黄バーで破壊される視覚的フィードバック。ジャンプの頂点で紫バーが生成されるタイミングが明確。 |
| (3) リスク・リワード | 高いジャンプでより多くのスパイクを育てられるが、スパイクが多いほど危険。ボーナス取得でスパイクを破壊できるが、タイミングが重要。 |
| (4) 斬新なメカニクス | ジャンプで「武器（紫バー）」を生成し、スパイクを育てる→ボーナスで「破壊武器（黄バー）」を生成し、育てたスパイクを破壊する二段階システム。フィールド自体を変化させる独特の攻略法。 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #weapon-change_field | ジャンプで紫バーを生成し、スパイクを育ててフィールドを変化させる | プレイヤー自身がフィールドの危険度をコントロールする戦略性を生む |
| #weapon-vertical | 上から下へ落ちるバーで攻撃 | 縦方向の攻撃により、回転する車輪上の任意の位置を狙える |
| #field-spike | 車輪の外周にスパイクが出現 | 常に存在する危険要素として緊張感を生み出す |
| #field-rotate | 車輪が回転し、スパイクとボーナスの位置が変化 | タイミングの判断と予測を要求する |
| #on_pressed-jump | タップでジャンプ、長押しで重力調整 | ジャンプの高さと滞空時間をコントロールし、バー生成のタイミングを調整 |
| #on_pressed-shoot | ジャンプの頂点で紫バーを自動生成 | ジャンプそのものが攻撃手段となる一石二鳥のメカニクス |
| #on_got_item-power_up | ボーナス取得で黄バーを生成 | スパイク破壊の機会を提供し、高得点への道を開く |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 車輪を惑星や時計に変更し、テーマを宇宙や時間に |
| Combine (結合) | 複数の車輪が異なる速度で回転し、全ての車輪でスパイクを管理 |
| Modify (修正) | 長押し時間でバーのサイズが変化し、より大きなスパイクを育てられる |
