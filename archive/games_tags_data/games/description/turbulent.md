# TURBULENT (turbulent)

**タグ**: #field-roughness, #on_pressed-jump, #on_holding-thrust, #rule-physics, #field-water, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す：ジャンプ / 長押し：推力で落下速度を遅くする |
| 挙動 | 波状に変動する水面の上を移動する船を操作。タップでジャンプし、着水時に飛距離に応じてスコアを獲得 |
| 終了条件 | 水面上を滑る地雷(*)に衝突 |
| スコア | ジャンプ開始地点からの水平距離の平方根に応じて加算 |
| 難易度 | 時間経過で波の振動速度が増し、地雷の出現間隔が短くなる |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| ship | bar(3,2), bar(4,2), bar(1,2) | 水面上では波の傾斜に沿って加速・減速。ジャンプ中は重力と推力の物理演算で移動 | - |
| mine(*) | text("*") 赤色 | 水面の起伏に沿って滑りながら移動。傾斜の影響を受けて加減速 | 終了 |
| waves | line 青色 | 各波が独立したサイン波で上下変動。画面左に自動スクロール | ジャンプ中に着水すると飛距離スコア獲得 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 青い波、赤い船と地雷という明確な色分け。波の起伏が視覚的に移動の難易度を示す。テキストなしで「波を飛び越えて地雷を避ける」というルールが直感的 |
| (2) 視覚的フィードバック | ジャンプ時の効果音、着水時のスコア表示と「hit」「powerUp」音。船の角度が移動方向を示し、地雷との接触が即座にゲームオーバー。危険が常に可視化 |
| (3) リスク・リワード | 大きくジャンプするほど高スコアだが、地雷への接触リスクも増大。長押しで滞空時間を調整し、安全な着地点を狙う判断が重要 |
| (4) 斬新なメカニクス | 波の起伏が移動速度に影響する物理演算。地雷も波に沿って滑る動きをするため、水面の形状が攻略の鍵となる独自性 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-roughness | 波状に変動する水面が移動速度と地雷の動きに影響 | 水面の起伏を読んで加速・減速を予測し、ジャンプタイミングを調整する技術が求められる |
| #on_pressed-jump | タップで水面から飛び出し、慣性と重力の影響下に移動 | ジャンプの開始位置と水面の傾斜が飛距離に影響。スコア最大化には波の形状を見極める判断力が必要 |
| #on_holding-thrust | ジャンプ中に長押しで上向きの推力を得て落下速度を減少 | 滞空時間を延ばして地雷をやり過ごす、または早く着水してスコアを稼ぐかの選択。状況判断が生存とスコアを左右 |
| #rule-physics | 船と地雷が波の傾斜に応じて加速・減速する物理演算 | 水面の形状が動的な障害物となり、同じ地雷でも波の状態で難易度が変化。物理の理解が攻略を深める |
| #field-water | 波状に変動する水面が移動の基準面となる | 不規則な水面が移動の予測を困難にし、常に変化する環境への適応力が試される |
| #field-auto_scroll | 画面が自動的に左にスクロールし続ける | 一定のペースで前進を強制され、タイミングを逃すと選択肢が狭まる。テンポ感が緊張を生む |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 水面を溶岩や毒の川に置き換え、接触自体がダメージ源に。または空中の雲の上を飛び移るゲームに変更 |
| Combine (結合) | アイテム収集要素を追加。水面上にコインを配置し、リスクを冒して取りに行くか無視するかの選択を加える |
| Modify (修正) | 波の高さや速度をランダムではなく、プレイヤーのスコアに応じて制御。ハイスコア時に穏やかな波でボーナスステージ化 |
