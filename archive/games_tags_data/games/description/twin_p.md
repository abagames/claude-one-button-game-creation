# TWIN P (twin_p)

**タグ**: #player-bar, #player-rotate, #on_pressed-reverse_state, #on_holding-extend

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し：線を伸ばす / 離す：固定点を反対側に切り替えて回転 |
| 挙動 | 赤と青の2つのP(ピン)が黒い線で結ばれて回転。長押しで線を伸ばし、離すと固定点が入れ替わる。線で黄色の○を取るとスコア獲得 |
| 終了条件 | 紫の×に線が触れる、または両方のピンが画面外に出る |
| スコア | 黄色の○を線で取ると、現在のマルチプライヤー分のスコア獲得。○を取る度にマルチプライヤー+1 |
| 難易度 | 時間経過で回転速度と線の伸縮速度が上昇し、オブジェクトの移動速度と出現頻度も増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| pins(P) | text("P") 赤/青 | 一方を固定点として回転。離すと固定点が入れ替わり、自動的に回転し続ける | - |
| line | line(2) 黒色 | 2つのピンを結ぶ線。長押しで最大99まで伸長。離すと15に戻る | - |
| circle(o) | box(4) 黄色 | 画面外から出現し、中央付近に向かって移動。画面外に出ると消失 | 線と接触で マルチプライヤー分のスコア + マルチプライヤー+1 |
| cross(x) | bar(4,2) × 2 紫色 | 画面外から出現し、中央付近に向かって移動。○よりも稀に出現 | 線と接触で終了 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 赤・青のピン、黒い線、黄色の○、紫の×という色分けが明確。回転する線で○を集め×を避けるというルールが直感的 |
| (2) 視覚的フィードバック | ○取得で「coin」音とスコア表示。×接触で「lucky」音と終了。長押し中は「laser」音が継続。視覚と聴覚で状態が明確 |
| (3) リスク・リワード | 線を長く伸ばすほど○を取りやすいが、×に当たるリスクも増大。マルチプライヤーシステムで連続取得に高い報酬 |
| (4) 斬新なメカニクス | 固定点が入れ替わる回転システム。線の長さ調整とタイミングで○を狙い、×を避ける独特の操作感 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-bar | 2つのピンを結ぶ線が当たり判定とアクション範囲を持つ | 線の位置と角度を予測し、○の軌道に合わせる技術が必要。線の両端が画面内に留まるよう管理 |
| #player-rotate | 固定点を中心に自動的に回転し続ける | 回転の軌道計算が攻略の核心。○が来るタイミングに線を合わせる予測力が求められる |
| #on_pressed-reverse_state | 離すと固定点が反対側のピンに切り替わり、180度回転 | 切り替えのタイミングで軌道を大きく変更可能。危険回避や○取得の位置調整に使う戦略的選択 |
| #on_holding-extend | 長押しで線が最大99まで伸長 | 伸ばすほど○を取りやすいが×のリスクも増加。また画面外に出やすくなる。リスク管理が重要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ピンの代わりに磁石、線の代わりに磁力場で○を引き寄せる。または線を鎖に変え、物理演算で揺れる挙動を追加 |
| Combine (結合) | 時間制限要素を追加。一定時間内に○を連続で取らないとマルチプライヤーがリセット。緊張感を高める |
| Modify (修正) | ピンの数を3つ、4つと増やし、複数の線が同時に回転。複雑な軌道と当たり判定でパズル性を強化 |
