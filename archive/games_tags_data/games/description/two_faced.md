# TWO FACED (two_faced)

**タグ**: #field-multiple, #player-follow, #player-inverted, #on_got_item-spawn, #player-rotate, #on_holding-stop, #on_holding-move

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップ：回転方向を反転 / 長押し：直進（回転停止） |
| 挙動 | 左右2つのエリアを行き来する蛇状のプレイヤー。離すと自動回転し、長押しで直進。アイテムを尾で取るとスコア獲得し尾が伸びる |
| 終了条件 | 頭が自分の尾に衝突 |
| スコア | アイテム(a)を取得すると、現在のマルチプライヤー分のスコア獲得 |
| 難易度 | 時間経過で移動速度が徐々に上昇。マルチプライヤーは時間で減少し、アイテム取得で大幅増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| head | bar(3,4) 緑色 | 離すと自動回転し、タップで回転方向反転。長押しで直進。画面端で反対側のエリアに転送 | 尾に当たると終了 |
| tails | bar(4,3) 薄緑/薄黒 | 頭の軌跡を追従。最大25セグメント。アイテム取得で+1 | - |
| item(a) | char("a") 黒色 | 各エリアに1つ出現。取得すると反対側のエリアにランダム再配置 | 頭または尾に触れるとマルチプライヤー × 尾の長さ分を加算 |
| areas | box(80) 薄黄/薄紫 | 左右2つのエリア。プレイヤーとアイテムはどちらか一方にのみ存在 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 2つのエリア（黄色と紫）と緑の蛇、黒いアイテムという色分け。蛇ゲームの要領でアイテムを集め、自分に当たらないようにするルールが直感的 |
| (2) 視覚的フィードバック | アイテム取得で「powerUp」音とスコア表示。タップで「laser」音、長押しで「hit」音。尾が伸びる様子とマルチプライヤー表示で成長が可視化 |
| (3) リスク・リワード | 尾が長いほど高スコアだが、自己衝突のリスクも増大。マルチプライヤーの時間減衰が素早いアイテム取得を促す |
| (4) 斬新なメカニクス | 2つのエリア間を転送される仕組み。アイテムが反対側に出現することで、エリア移動を強制する独自性 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-multiple | 左右2つのエリアに分かれたフィールド。画面端で反対側に転送 | エリア間の移動タイミングを計算し、反対側の状況を予測する空間認識力が必要 |
| #player-follow | 尾のセグメントが頭の軌跡を追従 | 過去の軌道が障害物となり、将来の移動範囲を制限。長期的な経路計画が求められる |
| #player-inverted | エリア移動時に回転方向が反転 | 移動後の操作感が変わるため、適応力が試される。混乱を利用した難易度設計 |
| #on_got_item-spawn | アイテム取得時に反対側のエリアにランダム再出現 | アイテムを取るとエリア移動が必須となり、リスクを冒して移動するか判断が求められる |
| #player-rotate | 離すと自動的に回転し続ける | 回転の軌道計算が攻略の核心。長押しで直進することで軌道を制御する選択 |
| #on_holding-stop | 長押しで回転を停止し直進 | 危険回避や正確なアイテム取得のための精密操作。停止判断がスコアと生存を左右 |
| #on_holding-move | 長押しで速度が上昇 | 素早くアイテムを取るか、慎重に動くかの選択。マルチプライヤー減衰との兼ね合い |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 2つのエリアを上下配置に変更。または3つ、4つのエリアに増やし、より複雑な転送システムを構築 |
| Combine (結合) | 障害物を追加。各エリアに移動する壁や敵を配置し、アイテム取得と障害物回避の両立を要求 |
| Modify (修正) | 尾の長さ上限を撤廃し、無限に伸びるように。または尾が時間で徐々に短くなり、アイテム取得で維持する仕組み |
