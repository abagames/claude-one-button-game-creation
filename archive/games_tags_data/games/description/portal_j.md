# PORTAL J (portal_j)

**タグ**: #on_pressed-jump_into_another, #field-holes, #field-1D

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップでポータルジャンプ |
| 挙動 | 床の上を自動で進み、タップすると地中に潜ってポータル位置に転送される |
| 終了条件 | 穴に落下するか、画面外に出る |
| スコア | ポータルまでの距離^1.5 × 0.1 × multiplier、ゴールド取得でmultiplier+1 |
| 難易度 | スクロール速度と落下速度が増加、床の移動速度が上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | box(5, 5) | 床の上を自動で進む、重力で落下 | - |
| portal | box(7, 5) | 右端から出現し、一定距離で次のポータルに切り替わる | タップでここに転送 |
| floor | rect(width, 8) | 横スクロール、一部は左右に移動、互いに反射 | プレイヤーを支える |
| gold | text("$") | 横スクロールで流れる | +multiplier |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 横スクロールの床と穴、ポータルという明快な構成。赤いプレイヤー、緑の床、紫のポータル、黄色のゴールドと色分けが明確。 |
| (2) 視覚的フィードバック | ポータルジャンプ時の地中への潜り込みと浮上のアニメーション、穴への落下の視覚化が分かりやすい。 |
| (3) リスク・リワード | 遠くのポータルまで待つほど高得点だが、穴に落ちるリスク増。ゴールド収集とポータルジャンプのタイミングの両立が求められる。 |
| (4) 斬新なメカニクス | ジャンプではなく「地中を潜って転送」という独特の移動方法。床同士が反射する物理演出が予測不可能性を生む。 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #on_pressed-jump_into_another | タップで地中に潜り、ポータル位置に転送される | 通常のジャンプとは異なる「別の場所への移動」を実現し、距離計測によるスコアリングを可能にする |
| #field-holes | 床と床の間に穴があり、落下するとゲームオーバー | ポータルジャンプのタイミングを強制し、緊張感を生み出す |
| #field-1D | 横スクロールの1次元フィールド | シンプルな移動軸により、タイミングの判断に集中できる |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ポータルをワープホールや時空の裂け目に変更し、SFテーマに |
| Combine (結合) | 複数のポータルが同時に存在し、色で区別して選択的にジャンプ |
| Modify (修正) | 長押し時間でポータルジャンプの飛距離を調整可能にする |
