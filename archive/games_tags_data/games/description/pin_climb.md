# PIN CLIMB (pin_climb)

**タグ**: #field-pins, #player-bar, #on_holding-extend, #player-rotate, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し（紐を伸ばす） |
| 挙動 | ピンに固定された紐が自動で回転し、長押しで伸びる。紐が他のピンに接触すると、そのピンに移動して固定 |
| 終了条件 | 現在のピンが画面下端に到達 |
| スコア | ピン間の距離（四捨五入）に応じて加算 |
| 難易度 | difficultyに応じて回転速度、伸びる速度、スクロール速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| cord | line(black) | ピンから伸びる紐、自動で回転（角度+=difficulty×0.05）、長押しで伸びる（長さ+=difficulty） | - |
| pins | box(3, black) | ランダムな位置に配置、紐の固定点 | 紐が接触すると移動し、新しい固定点になる |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | ピンと紐だけのミニマル構成、黒一色のシンプルなビジュアル |
| (2) 視覚的フィードバック | ピン移動時の効果音とスコア表示、紐の長さと角度が視覚的に明確 |
| (3) リスク・リワード | 紐を長く伸ばせば遠いピンに届き高得点だが、タイミングが難しくなる。回転速度が速いほど判断時間が短い |
| (4) 斬新なメカニクス | 回転+伸縮+ピン移動の組み合わせ。距離が直接スコアになる設計が、リスクテイクを促す |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-pins | 複数のピンが画面上に配置され、それぞれが紐の固定点として機能 | ピンの配置を予測し、次に移動するピンを計画する必要がある |
| #player-bar | 紐がピンから伸びる棒として表現される | 棒の長さと角度を制御し、目標のピンに接触させる必要がある |
| #on_holding-extend | 長押しで紐が伸びる（長さ+=difficulty） | 伸ばすタイミングとリリースのタイミングが重要。早すぎても遅すぎても目標に届かない |
| #player-rotate | 紐が自動で回転（角度+=difficulty×0.05） | 回転のタイミングを見計らって、目標のピンの方向で紐を伸ばす必要がある |
| #field-auto_scroll | 下方向に自動スクロール、現在のピンの高度で速度調整 | 時間制限的なプレッシャー、画面下端に到達する前に次のピンに移動する必要がある |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ピンを「星」に置き換え、宇宙空間を渡り歩くゲームに変化させる |
| Combine (結合) | 複数の紐を同時に制御する要素を追加し、より複雑な操作を要求する |
| Modify (修正) | 回転速度をボタン長押しで制御可能にし、より精密なタイミング調整を可能にする |
