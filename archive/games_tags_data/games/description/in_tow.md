# IN TOW (in_tow)

**タグ**: player-follow, field-auto_scroll, field-floors, on_got_item-spawn

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | タップでジャンプ、空中でひよこを1羽消費して追加ジャンプ可能 |
| 終了条件 | 鳥が弾丸に当たりひよこが0羽、または画面下に落下 |
| スコア | 床のひよこを取得すると得点（所持ひよこ数に応じて増加） |
| 難易度 | 時間経過で弾丸の速度と出現頻度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| bird | キャラクター（上昇/下降） | タップでジャンプ、重力で落下 | - |
| chick | キャラクター（追従） | 鳥の軌跡を3フレーム間隔で追従 | 床で取得時+得点、弾丸で落下 |
| bullet | キャラクター（矢印） | 右から左へ移動 | 鳥に当たるとひよこ全滅 |
| floor | 黄色の横バー | 右から左へスクロール | 鳥が着地可能 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | キャラクターの追従動作が視覚的に分かりやすく、ひよこの列が戦略資源を表現 |
| (2) 視覚的フィードバック | ひよこの軌跡追従、弾丸衝突時の落下アニメーション |
| (3) リスク・リワード | 多くのひよこ所持で追加ジャンプ増加だが、弾丸に当たると全滅のリスク |
| (4) 斬新なメカニクス | ひよこ消費の追加ジャンプ、軌跡追従システム、リソース管理とジャンプ回数の関連 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| player-follow | ひよこが鳥の軌跡を追従 | ひよこの配置を意識した軌道選択、弾丸との接触面積を管理 |
| field-auto_scroll | 画面が右から左へスクロール | 継続的な前進を強制、弾丸と床のタイミング判断が必要 |
| field-floors | 複数の床が配置 | 床の高さを利用したジャンプタイミングの調整 |
| on_got_item-spawn | ひよこ取得で所持数増加 | ひよこ数がジャンプ回数とスコア倍率に直結、取得タイミングの判断 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ひよこを消費してシールド展開、弾丸を一定時間防御 |
| Combine (結合) | 異なる色のひよこを追加し、色ごとに異なる能力付与 |
| Modify (修正) | タップ長押しでひよこ全員を前方に射出し、弾丸を相殺 |
