# AERIAL BAR (aerial_bar)

**タグ**: player-bar, on_pressed-jump_into_another, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップで離脱 / ホールドで上昇制御 |
| 挙動 | 天井からぶら下がる回転バーを渡り歩き、飛行時間を稼いでスコアを獲得 |
| 終了条件 | バーまたはプレイヤーが左端到達 / バーが床接触 / プレイヤーが天井接触 |
| スコア | バー間の飛行時間に応じて加算 (flyingTicks) |
| 難易度 | 時間経過でスクロール速度上昇、バーの揺れ加速、飛行速度増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | bar(10,4) | バーにぶら下がり回転、タップで飛行 | - |
| bars | line+box(5) | 天井から吊り下がり重力で揺れ動く | 接触で掴まる |
| ceiling | rect(100,変動) | 着地失敗で上昇 | 接触で反発 |
| floor | rect(100,10) | 下部に固定 | 接触で大跳ね返り |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 青いバーと緑の棒だけのミニマル構成。バーの揺れで次の動きが予測可能 |
| (2) 視覚的フィードバック | 掴んだバーは黒に変化。飛行中の軌跡で成功/失敗が明確 |
| (3) リスク・リワード | より長く飛ぶほど高得点だが、着地失敗時は天井が上昇してスペースが狭くなる |
| (4) 斬新なメカニクス | バーの物理的な振り子運動と遠心力を利用した飛行。ホールドで上昇補助という二段階入力 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| player-bar | プレイヤーが棒状で回転・揺れる | バーの角度と回転速度が飛行の初速を決定。タイミングが重要 |
| on_pressed-jump_into_another | タップで別のバーへ飛び移る | 最適な離脱タイミングを見極める判断力が求められる |
| field-auto_scroll | 画面が自動スクロール | 常に前進を強制され、躊躇する時間がない緊張感を生む |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | バーを蔓や鎖に変え、伸縮する要素を追加 |
| Combine (結合) | 複数のプレイヤーが協力して同じバーを掴む協力モード |
| Modify (修正) | バーの長さや揺れ速度をランダム化、または色で予告 |
