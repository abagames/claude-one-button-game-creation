# BALL TOUR (ball_tour)

**タグ**: player-reciprocate, on_holding-move

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | ホールドで前進加速 / 離すと減速 |
| 挙動 | サイン波で自動的に上下するキャラクターを操作。緑のボールを集め、黒いスパイクを避ける |
| 終了条件 | 頭上の判定が黒いスパイク"*"に接触 |
| スコア | 緑のボールを取得するとmultiplier+10でスコア獲得。時間経過でmultiplier-0.02ずつ減少 |
| 難易度 | 時間経過でサイン波の振動速度と横スクロール速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("ab") | サイン波で自動上下移動 | - |
| head | char("c"赤) | プレイヤー頭上の判定 | スパイク接触で終了 |
| spikes | text("*") | 右へ流れる、一部は上下移動 | 接触で終了 |
| balls | char("c"緑) | 右へ流れる | +multiplier、スコア獲得 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 上下に揺れる動きが視覚的に明快。緑=良い、黒=悪い、赤=危険の色分け |
| (2) 視覚的フィードバック | サイン波の動きでリズム感。multiplierの数値で緊張感の可視化 |
| (3) リスク・リワード | ホールドで速く進むほどボールを多く集められるが、回避も困難に |
| (4) 斬新なメカニクス | サイン波による強制的な往復運動。速度制御だけでタイミングを調整する制約 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| player-reciprocate | プレイヤーが自動的に往復運動 | 直接的な位置制御ができず、タイミングと速度で間接制御する必要 |
| on_holding-move | ホールドで前進速度が上昇 | 速度調整がゲームの主要操作。リズムゲーム的な要素 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | サイン波を別の波形(三角波、矩形波)に変え、動きの予測可能性を調整 |
| Combine (結合) | 左右の揺れも追加し、2次元のリサジュー図形で移動 |
| Modify (修正) | ホールド時間でサイン波の振幅を変更し、より直接的な制御を追加 |
