# LLAND (lland)

**タグ**: #on_holding-thrust, #field-roughness, #field-spike

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | 月面着陸船を操作し、不規則な地形の間を飛行しながら青い着陸地点に正確に着陸 |
| 終了条件 | 赤い山に衝突、または青い着陸地点を見逃すとゲームオーバー |
| スコア | 青い着陸地点に着陸するごとに+1点 |
| 難易度 | スクロール速度が時間経過で増加し、着陸の精度と反応速度が要求される |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| ship | char(a) | 重力で落下、Hold時に上昇推進＋パーティクル噴射 | - |
| red_mountain | rect(13x99) | 左にスクロール、高さはランダム（40-90または70-90） | 終了 |
| cyan_landing | rect(13x99) | 左にスクロール、約9山ごとに1回出現、高さ40-70 | +1点、次まで待機 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 赤＝危険、青＝着陸地点という色分けが明確。船から噴射されるパーティクルが推進力の視覚化として機能 |
| (2) 視覚的フィードバック | Hold時の「laser」音と下向きパーティクル、着陸時の「select」音とパーティクル、衝突時の「explosion」音と"X"マーク表示 |
| (3) リスク・リワード | 高い位置を保つと安全だが燃料（推進力）を多用、低い位置は危険だが青い着陸地点に素早く反応可能 |
| (4) 斬新なメカニクス | 着陸後は次の推進まで待機する独特のリズム。連続スコアリングではなく、1回ずつの緊張と解放のサイクル |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #on_holding-thrust | 長押しで上昇推進、離すと重力で落下 | 推進力の微調整が着陸精度を左右。過剰な推進は画面上端への衝突リスク |
| #field-roughness | 山の高さが不規則に変化し、安全な飛行高度が常に変動 | 地形の凹凸を先読みし、適切な高度を維持する判断力が必要 |
| #field-spike | 赤い山が即死の障害物として左から迫る | 着陸地点に集中しすぎると赤い山を見落とす。視野の広さが求められる |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 重力を横方向に変更し、洞窟の天井と床を避けながら進む横スクロールに |
| Combine (結合) | 着陸地点に時間制限を追加し、一定時間内に着陸しないと消失するプレッシャー要素 |
| Modify (修正) | 燃料ゲージを追加し、推進力に制限を設ける。着陸時に燃料補給で資源管理要素を強化 |
