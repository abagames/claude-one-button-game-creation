# WIPER (wiper)

**タグ**: #weapon-wipe, #on_pressed-reverse_state, #obstacle-fall, #field-outpost

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | 画面下部の拠点から自動往復するワイパーを操作し、ボタンを押すとワイパーを目標位置へ移動させ往復方向を反転 |
| 終了条件 | 雨粒が画面下端（y=99）に到達 |
| スコア | ワイパーで雨粒を拭き取ると得点（サイズが大きいほど高得点） |
| 難易度 | 雨の出現頻度が増加、落下速度も上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| wiper | bar(36,6) | 拠点(50,110)を中心に左右に往復運動、端で自動反転 | - |
| rain | box(√size) | 上から落下、下の雨粒に接近すると合体して大きくなる、サイズに応じて落下速度増加 | ワイパーで拭き取られると消滅＋得点 |
| outpost | bar(36,2) | ワイパーの目標位置を示す影、ボタンで切り替わる | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | ワイパーの往復運動と薄い影で次の位置が予測可能。雨粒の合体による成長が視覚的に分かりやすい。シアン色の雨粒と黒いワイパーのコントラストが明確 |
| (2) 視覚的フィードバック | 雨粒が合体する際の音、拭き取り時の音とスコア表示で成功が明確。画面下端到達時は赤い「X」マークで失敗を強調 |
| (3) リスク・リワード | 大きく成長した雨粒は高得点だが落下速度も速く、拭き取りタイミングが難しい。複数の雨粒を合体させてから拭き取る戦略がハイリスク・ハイリターン |
| (4) 斬新なメカニクス | 雨粒の合体システムがユニーク。小さい雨粒を意図的に残して大きく育てる戦略と、早めに処理する安全策の選択が戦略的 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #weapon-wipe | ワイパーで接触した雨粒を拭き取る | 往復運動の軌道上に雨粒を配置する位置取りが重要。ワイパーの到達範囲を常に意識 |
| #on_pressed-reverse_state | ボタンで往復方向を反転＋目標位置へ瞬間移動 | タイミング良く反転させて危険な雨粒を優先処理。予測的な操作で効率的に拭き取る |
| #obstacle-fall | 雨粒が上から落下、サイズに応じて速度変化 | 落下軌道を予測してワイパーの往復タイミングを調整。大きい雨粒ほど緊急度が高い |
| #field-outpost | 画面下部の拠点からワイパーを展開 | 拠点からの距離によって到達時間が変わる。遠くの雨粒への対処は計画的に |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 雨粒を「雪」に置き換え、積もると画面が狭くなるメカニクスを追加 |
| Combine (結合) | 複数のワイパーを同時操作し、異なる高度で雨粒を処理するパズル要素を導入 |
| Modify (修正) | ワイパーの往復速度を雨粒のサイズに応じて変化させ、大きい雨粒ほど高速で拭き取れるように |
