# JUMP ON (jump_on)

**タグ**: #on_pressed-select_route, #obstacle-chase, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（穴から上昇中のみ） |
| 挙動 | 穴に自動飛び込み→下降→上昇、タップで任意のタイミング・レーンに飛び出す |
| 終了条件 | 敵に追いつかれる、または画面左端に到達 |
| スコア | 敵が残した黄色い点を取得、倍率が連続で上昇 |
| 難易度 | 時間経過で敵の出現頻度と移動速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | キャラクター（黒） | 自動で穴に飛び込み、タップで飛び出してレーン選択 | 敵と接触で終了 |
| enemy | キャラクター（赤） | 自動で穴に飛び込み、プレイヤーと同じ高さで飛び出す、歩行中に点を残す | - |
| hole | 白い縦バー | プレイヤーと敵が飛び込む、アニメーション付き | - |
| dots | 黄色い点 | 敵の歩行軌跡 | プレイヤーが取得で+得点、倍率+1 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | キャラクターと色分け、穴への飛び込み・飛び出しの視覚的パターン |
| (2) 視覚的フィードバック | 穴のアニメーション、点の配置で敵の軌跡を可視化 |
| (3) リスク・リワード | 敵を避けつつ点を取る経路選択、穴からの飛び出しタイミングの判断 |
| (4) 斬新なメカニクス | 穴を使った垂直移動とレーン選択、追跡敵との競争システム |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| on_pressed-select_route | 穴から飛び出すタイミングでレーン選択 | 敵の位置と点の配置を見極め、最適なレーンと高さを選択 |
| obstacle-chase | 敵が後方から追跡 | 画面左端到達を避けつつ、敵との距離を管理 |
| field-auto_scroll | 画面が右から左へスクロール | プレイヤー位置に応じてスクロール速度が変化、前進の維持が必要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 穴を横方向のワープゾーンに変え、左右のレーン移動に |
| Combine (結合) | 点に色を追加し、色に応じた倍率変化や特殊効果 |
| Modify (修正) | タップ長押しで穴内の滞在時間を延長、敵をやり過ごす戦略 |
