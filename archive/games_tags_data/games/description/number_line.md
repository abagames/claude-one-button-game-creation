# NUMBER LINE (number_line)

**タグ**: #rule-mathematics, #rule-time_limit

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（画面内の数字を合算） |
| 挙動 | 青い線と赤い線があり、それぞれの線上を1-9の数字が左から右へ流れる。ボタンを押すと画面内の数字が合算され、約60フレーム後に（青の合計-赤の合計）の差分に応じて線が移動 |
| 終了条件 | 青い線と赤い線が接触 |
| スコア | 青い線が上端に達すると1点ずつ加算、合算時の差分（青-赤）がスコアに加算 |
| 難易度 | difficultyに応じて線の移動速度、数字の出現頻度、合算後の線の移動量が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| numbers | text(1-9, 色分け) | 青い線と赤い線上を左から右へ移動、合算時は画面中央に集まって合計値を表示 | - |
| blue_line | rect(blue, 1px) | 画面上部、合算結果が正なら下降、負なら上昇 | 赤い線と接触で終了 |
| red_line | rect(red, 1px) | 画面下部、合算結果が正なら上昇、負なら下降 | 青い線と接触で終了 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 2本の線と数字だけのミニマル構成、数字の色分け（青系/緑系/紫系/赤系）で視認性向上 |
| (2) 視覚的フィードバック | 合算時は数字が中央に集まる演出、合算結果で線が移動する方向が視覚的に明確、効果音で成功・失敗を通知 |
| (3) リスク・リワード | 青の数字が多いタイミングで合算すれば高得点だが、タイミングを逃すと赤が増えて不利に。待ちすぎると線が接触するリスク |
| (4) 斬新なメカニクス | 引き算という基本的な算数と、タイミングゲームの融合。2つの線の位置が戦況を視覚化し、逆転の可能性を常に示唆 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #rule-mathematics | 青い線と赤い線上の数字を合算し、その差分（青-赤）でゲームの進行が決まる | 暗算能力とタイミング判断が同時に求められる。数字の組み合わせを瞬時に計算し、最適な合算タイミングを見極める必要がある |
| #rule-time_limit | 合算後約60フレーム（難易度で変動）で結果が確定し、線が移動。線が接触するまでの時間制限 | 焦らず冷静に数字を見極める必要があるが、時間制限が判断を急がせる緊張感を生む |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 引き算を掛け算や割り算に変え、より複雑な数学パズルに変化させる |
| Combine (結合) | 3本以上の線を追加し、複数の計算を同時に管理する要素を加える |
| Modify (修正) | 負の数字を導入し、戦略的に負の数字を利用する要素を追加 |
