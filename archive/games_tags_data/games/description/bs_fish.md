# BS FISH (bs_fish)

**タグ**: field-water, on_holding-move, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | 鳥が水平方向に飛行、長押しで加速して魚を捕獲する |
| 終了条件 | 大きな白い魚（障害物）に衝突 |
| スコア | 魚を取得すると現在の倍率(multiplier)分の得点、通常魚で倍率+1、目玉魚で倍率+10 |
| 難易度 | 時間経過で魚の速度と出現頻度が増加、鳥の最高速度も上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| bird | char("b") アニメーション | 水平方向に移動、長押しで加速(最大3×difficulty)、通常は0.1の速度 | - |
| fish (normal) | char("a") 黒 | 上から放物線を描いて落下、水面(y=50)で青色に変化 | +multiplier点、倍率+1 |
| fish (eye) | char("a") 青 | 通常魚と同じ動作 | +multiplier点、倍率+10 |
| fish (big) | char("a")×6 + 白い線 | 通常魚より大きく、速度が異なる | ゲームオーバー |
| fish (fake) | char("a")×6 | 水面に到達すると13個の小魚（1個は目玉魚）に分裂 | 分裂のみ |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 鳥と魚という分かりやすいモチーフ、色分け（黒=空中、青=水中）で状態を表現、大きさで危険度を示す |
| (2) 視覚的フィードバック | 水面の青いライン、魚の色変化、倍率表示(x{multiplier})、大きな魚の白い線による強調 |
| (3) リスク・リワード | 加速すると魚を多く取れるが制御が難しくなる、倍率を維持するために魚を逃さないようにする緊張感、偽魚の分裂による高リスク高リターン |
| (4) 斬新なメカニクス | 偽魚の分裂システム（1個が13個に）、倍率システム（取得で増加、逃すと減少）、速度制御による難易度調整 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| field-water | 水面(y=50)で魚の色が変化、視覚的な領域区分 | 水面上下で魚の取得タイミングを判断、水面での偽魚分裂に注意が必要 |
| on_holding-move | 長押しで加速(最大3×difficulty)、離すと減速(0.1) | 速度制御による位置調整、高速移動でのリスク管理、難易度に応じた速度適応 |
| field-auto_scroll | 画面が自動的に右方向にスクロール | 一定のペースで進行、後退不可による緊張感、倍率維持の時間的プレッシャー |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 鳥を魚に、魚を鳥に入れ替えて水中探索ゲームに / 倍率システムをコンボカウンターに変更 |
| Combine (結合) | 武器システムを追加して大きな魚を撃破可能に / 複数の水面レイヤーで立体的な動き |
| Modify (修正) | 重力方向を変えて上昇する魚を追う / 倍率が下がるペナルティを強化して緊張感を高める / 偽魚の分裂パターンをランダム化 |
