# NS CLIMB (ns_climb)

**タグ**: #on_pressed-reverse_state, #field-gravity, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（N極とS極を切り替え） |
| 挙動 | 左右の壁に配置された磁石から磁力を受ける。同極は反発、異極は吸引。重力により下方向に加速 |
| 終了条件 | 画面下端に到達 |
| スコア | 上昇した距離に応じて加算 |
| 難易度 | difficultyに応じて重力と磁力が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | box(7,7, red/blue) | N極(赤)またはS極(青)の極性を持ち、磁石から磁力を受けて移動 | - |
| magnets | box(9,9) | 左右の壁に11個ずつ配置、N極(赤)/S極(青)/無極性(灰)の3種類、プレイヤーに磁力を及ぼす | 壁に押し戻される |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 赤(N極)と青(S極)の2色で磁石の極性を表現、テキストで補足。磁力による物理挙動が直感的 |
| (2) 視覚的フィードバック | 極性は色とテキスト("N"/"S")で明示、衝突時の効果音で状態変化を通知 |
| (3) リスク・リワード | 磁石に接近しすぎると壁に衝突するリスク、しかし適切な極性なら反発力で上昇できる |
| (4) 斬新なメカニクス | 磁力の距離の2乗に反比例する物理法則を使った重力に逆らう上昇ゲーム。極性切り替えという単純操作から複雑な軌道制御が生まれる |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #on_pressed-reverse_state | ボタンを押すとN極とS極が入れ替わる | 磁石の配置を見て、吸引されるか反発されるかを瞬時に判断し、適切なタイミングで極性を変える必要がある |
| #field-gravity | 常に下方向への重力が働く | 重力に逆らって上昇し続ける緊張感、磁力と重力のバランスが戦略の核 |
| #field-auto_scroll | プレイヤーが上昇すると画面が自動スクロール | 一定の高度を維持することでスコアを稼ぎ、落下を防ぐ必要がある |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 磁石を「風」に置き換え、左右からの風向きをボタンで切り替える |
| Combine (結合) | 一時的に磁力を無効化するアイテムを追加し、複雑な磁石配置を安全に通過可能にする |
| Modify (修正) | 3種類以上の極性を導入し、ボタンを押すごとに循環させることで戦略の幅を広げる |
