# DESCENT S (descent_s)

**タグ**: on_holding-thrust, rule-physics, field-weather

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押しで上向き推力 |
| 挙動 | 長押しで上昇加速、離すと下降加速。デッキ着陸時の速度が1以下なら成功 |
| 終了条件 | 速度1超過で着陸、画面上端接触、着陸デッキ外に落下 |
| スコア | 着陸時のデッキ右端からの距離。遠いほど高得点 |
| 難易度 | 水平速度増加、上昇・下降加速度のランダム変化拡大 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| ship | char(a) | 慣性と重力で移動。長押しで上向き推力 | - |
| deck (黒) | rect | 右へ自動スクロール | 速度1以下で得点、超過でゲームオーバー |
| deck area (黄) | rect | デッキの下と右の領域 | 接触でゲームオーバー |
| velocity bar | rect | 左端に速度バー(青/黄/赤)。赤線が1.0の閾値 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 船と着陸デッキのみ。速度バーの色(青→黄→赤)で危険度が一目瞭然 |
| (2) 視覚的フィードバック | 左端の速度バーと赤線の閾値で着陸可否が明確。次のデッキの推力値が事前表示される |
| (3) リスク・リワード | 速く降下して遠くまで行くか、安全に早めに着陸するか。次の推力値の変化がリスク管理の鍵 |
| (4) 斬新なメカニクス | 着陸ごとに推力特性が変化し、次のデッキで適用。風速の変化を模した「次の天候」予報システム |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| on_holding-thrust | 長押しで上向き推力を発生 | 慣性制御の習熟。押し続ける時間と離すタイミングの両方が重要 |
| rule-physics | 加速度・速度・位置の物理計算 | リアルな慣性挙動による予測と制御。物理的直感が求められる |
| field-weather | 着陸ごとに推力値が変化 | 固定パターンの習熟ではなく、変化する条件への適応力を要求。次の条件の事前表示が戦略的判断を可能に |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 推力を「重力方向の切り替え」に置き換え、タップで上下の重力を反転。慣性はそのまま |
| Combine (結合) | field:auto_scrollの速度が着陸精度に応じて変化。完璧な着陸で減速、荒い着陸で加速 |
| Modify (修正) | 横風(水平加速度)を追加し、2軸の制御が必要に。次のデッキの風向きと強さも事前表示 |
