# SMILY ANGRY (smily_angry)

**タグ**: #obstacle-foresee, #on_pressed-turn

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | プレイヤーは画面下部を自動移動。タップで左右の移動方向を反転させ、笑顔キャラから発射されるボーナス弾を取得しつつ、怒り顔からの危険弾を回避 |
| 終了条件 | 怒り顔キャラから発射される弾に触れる |
| スコア | 笑顔キャラが発射するボーナス弾($)を取得すると multiplier に応じたスコアを獲得し、multiplier が1増加 |
| 難易度 | 時間経過でキャラの発射間隔が短縮され、弾速が上昇。プレイヤーの移動速度も上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char(c/d) | 画面下部を一定速度で左右に自動移動、タップで反転 | - |
| smile(赤) | char(a) 黄色 | ランダムに移動しつつ、プレイヤーの未来位置を予測してボーナス弾を発射 | - |
| smile(青) | char(a) 緑色 | ランダムに移動しつつ、プレイヤーの現在位置に向けてボーナス弾を発射 | - |
| angry(赤) | char(b) 赤色 | ランダムに移動しつつ、プレイヤーの未来位置を予測して危険弾を発射 | - |
| angry(青) | char(b) 水色 | ランダムに移動しつつ、プレイヤーの現在位置に向けて危険弾を発射 | - |
| bonus弾 | text($) | 発射元に向けて直進、画面下部で消失 | +multiplier点、multiplier+1 |
| 危険弾 | bar(3,2) | 発射元に向けて直進、画面下部で消失 | ゲームオーバー |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 笑顔と怒り顔のキャラクターで敵味方が一目瞭然。色で追跡性能（赤=予測、青=直接）が区別される。ボーナス弾($)と危険弾(bar)も視覚的に明確 |
| (2) 視覚的フィードバック | キャラの表情と色が動的に変化し、緊張感を演出。ボーナス取得時の効果音と multiplier 表示で達成感を強化 |
| (3) リスク・リワード | ボーナス弾を取り逃すと multiplier が減少するため、危険を冒してでも取得する価値がある。赤キャラの予測弾はより高難度だが避けられれば大きなアドバンテージ |
| (4) 斬新なメカニクス | 同じキャラが時間経過で敵味方を切り替える。赤キャラの「未来予測」による高度な追跡が、プレイヤーに戦略的な移動を要求 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #obstacle-foresee | 赤いキャラはプレイヤーの未来位置を予測して弾を発射 | 単純な直線移動では回避不可能。意図的な方向転換のタイミングが重要となり、より高度な読み合いが生まれる |
| #on_pressed-turn | タップで移動方向を左右に反転 | 連続タップによる細かい位置調整と、予測を外すための不規則な動きの使い分けが攻略の鍵 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | キャラを天候アイコン（晴れ/雨）に変え、雨を避けつつ太陽の光を浴びる天候ゲームにする |
| Combine (結合) | プレイヤーが弾を発射できる要素を追加し、怒り顔を撃って笑顔に変えるシューティング要素を加える |
| Modify (修正) | 3つ以上のキャラを配置し、それぞれ異なる発射パターン（扇状、波状など）を持たせて複雑化 |
