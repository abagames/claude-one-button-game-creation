# G PRESS (g_press)

**タグ**: obstacle-well_up, rule-time_limit, rule-gauge_management, on_pressed-attack

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（プレスの幅を縮小して攻撃） |
| 挙動 | 上下に動くプレスで、湧き上がる泡を潰す。大きい泡をドロップに分裂させ、水位を下げる |
| 終了条件 | 水位が100に達する |
| スコア | 水位を下げた量の sqrt(量) × ceil(量) 点 |
| 難易度 | 時間経過で泡の出現頻度と速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| press（プレス） | rect(5, 20) 2本 black/red | 上下に移動。タップで幅が縮小（0〜80）、その後ゆっくり拡大 | - |
| bubbles（泡） | box(size) purple | 水面から上昇（サイズ5〜9）。大きいほどゆっくり | プレス接触: サイズ>幅ならドロップ化、サイズ≤幅なら押し出し |
| drops（ドロップ） | box(2) light_purple | 泡が分裂して生成。重力で落下 | 水面到達: 水位-0.35、スコア加算 |
| water（水面） | rect(100, height) purple | 泡の出現で上昇、ドロップで下降 | - |
| danger zone | rect purple | プレス縮小中または幅0時に紫線で危険範囲を表示 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | プレスで泡を潰す、という分かりやすいメカニクス。水位ゲージが常に視覚化され、危機感が明確。紫色の危険範囲表示が直感的 |
| (2) 視覚的フィードバック | プレス幅の変化、泡の分裂、ドロップの落下、水位の上下。スコア加算時の演出。幅0時の警告音と色変化 |
| (3) リスク・リワード | プレスを縮めて大きい泡を分裂させると高得点だが、幅0になると危機。タイミングを見極める必要がある |
| (4) 斬新なメカニクス | プレス幅を動的に調整する独自システム。「泡を潰す」ではなく「泡を分裂させる」という逆転的アプローチ。水位管理とプレス幅管理の二重ゲージ制御 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| obstacle-well_up | 水面から泡が湧き上がり、水位が上昇 | 常に水位が上昇圧力を受け、時間制限的なプレッシャー。泡の出現パターンを読む習熟要素 |
| rule-time_limit | 水位が100に達するとゲームオーバー | 泡の処理速度と水位管理が重要。ドロップ化による水位低下とのバランスが鍵 |
| rule-gauge_management | 水位（ゲージ）を管理し、100未満に保つ | プレス幅と水位の二重ゲージ管理。どちらも適切に制御しないと失敗 |
| on_pressed-attack | タップでプレス幅を縮小し、大きい泡を攻撃 | プレス幅を縮めるタイミングが戦略的。大きい泡を狙って分裂させると高得点だが、幅0のリスクもある |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | プレスを「扇風機」に変更。タップで風を送り、泡を横に押し出す |
| Combine (結合) | プレス幅に応じて色が変わり、特定の色の泡のみ潰せる「色マッチング」要素を追加 |
| Modify (修正) | 泡のサイズバリエーションを増やし、巨大泡（10〜20）を追加。分裂時のボーナスを大幅に増加 |
