# PARKING (parking)

**タグ**: #player-multiple, #on_holding-thrust, #field-auto_scroll, #field-spike

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し（右に旋回）/ 離す（左に旋回） |
| 挙動 | 複数台の車を同時に操作し、右側の黄色い駐車スペースに駐車。長押しで右旋回（-90°→0°）、離すと左旋回（0°→-90°） |
| 終了条件 | 他の車や駐車済みの車に衝突、または画面下端に到達 |
| スコア | 駐車成功で10×multiplier点、お金アイテム($)取得でmultiplier点とmultiplier+1、お金を逃すとmultiplier-1 |
| 難易度 | difficultyに応じて車の速度と旋回速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| cars | bar(色) + box(black, タイヤ×4) | 複数台を同時操作、上から出現し前進、同じ角度で旋回 | 他の車や障害物と衝突で終了 |
| parking_space | rect(light_yellow) | 右側に固定された駐車スペース | 車が進入すると駐車成功、10×multiplier点 |
| parkedCars | bar(色) + box(black, タイヤ×4) | 駐車済みの車、障害物として機能 | 衝突で終了 |
| gold | box(yellow) + text("$") | ランダムな位置に出現、下へスクロール | 取得でmultiplier点とmultiplier+1 |
| spikes | box(light_black) | 左右の壁と道路のマーカー | 衝突判定なし（視覚的境界） |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 車の旋回と駐車というシンプルなルール、色分けで車と駐車スペースを区別 |
| (2) 視覚的フィードバック | 駐車成功/失敗の効果音、スコア表示、"PARKING"の文字で駐車スペースを明示 |
| (3) リスク・リワード | 複数台を同時操作することでスコアが増えるが、衝突リスクも増加。お金を取るために軌道を変えるリスク |
| (4) 斬新なメカニクス | 複数車両の同時操作という独特の難しさ。旋回角度の制限（-90°～0°）が戦略の核 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-multiple | 複数台の車を同時に操作、駐車成功するごとに台数が増加 | 複数車両の軌道を同時に予測し、すべての車が衝突せずに駐車するルートを計画する必要がある |
| #on_holding-thrust | 長押しで右旋回（角度+=0.1×√difficulty）、離すと左旋回（角度-=0.1×√difficulty） | 旋回のタイミングとリリースのタイミングが重要。駐車スペースに正確に進入する角度調整が求められる |
| #field-auto_scroll | 下方向に自動スクロール、車の位置で速度調整 | 時間制限的なプレッシャー、画面下端に到達する前に駐車する必要がある |
| #field-spike | 左右の壁と駐車済みの車が障害物として機能 | 狭い道路で複数車両を操作する空間的制約が戦略の核 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 車を「飛行機」に置き換え、着陸ゲームに変化させる |
| Combine (結合) | 各車両が異なる速度や旋回速度を持つ要素を追加し、個別の制御を要求する |
| Modify (修正) | 駐車スペースのサイズや角度を変化させ、より精密な操作を要求する |
