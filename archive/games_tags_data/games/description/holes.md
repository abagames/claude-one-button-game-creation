# HOLES (holes)

**タグ**: rule-physics, on_pressed-reverse_state, field-bottomless, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | タップで壁の穴配置パターンを切り替え、ボールを物理法則に従って落下させずに維持 |
| 終了条件 | すべてのボールが画面下に落下 |
| スコア | 画面上昇速度に応じた得点、コイン取得でボーナス（ボール数×10点） |
| 難易度 | 時間経過でボールの重力と速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| ball | 円（半径1） | 重力と慣性で移動、壁と他ボールで反射 | - |
| wall | 斜めバー（緑/シアン） | 2つのindexパターンで表示切替、画面下から上へ流れる | ボールを反射 |
| coin | 黄色ライン | 壁に付随して配置 | +得点、ボール追加、新しい壁生成 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 物理演算の直感性、色分け（緑/シアン）で2つのパターンを識別 |
| (2) 視覚的フィードバック | ボールの放物線運動、壁の表示/非表示切替で状態変化を明示 |
| (3) リスク・リワード | コイン取得でボールが増え高得点だが、多数のボール管理が複雑化 |
| (4) 斬新なメカニクス | 穴配置の2パターン切替による物理パズル、複数ボールの同時管理 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| rule-physics | 重力と反射の物理演算 | ボールの軌道予測と壁の角度を利用した制御が必要 |
| on_pressed-reverse_state | タップで壁の穴パターン切替 | 適切なタイミングでパターンを切り替え、ボールを穴に導く判断 |
| field-bottomless | 画面下に落ちるとボール消失 | すべてのボールを維持する緊張感 |
| field-auto_scroll | 画面が上方向に自動スクロール | ボールを上部に保つための継続的な操作が必要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 重力を横方向に変え、左右のスクロールゲームに |
| Combine (結合) | 壁の穴パターンを3つ以上に増やし、数字キーで選択 |
| Modify (修正) | タップでボール自体の重力方向を反転させる操作に変更 |
