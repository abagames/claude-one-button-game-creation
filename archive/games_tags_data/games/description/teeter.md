# TEETER (teeter)

**タグ**: #rule-classify, #on_pressed-reverse_state, #rule-physics, #field-pins

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップでバー角度反転（全バー同時） |
| 挙動 | ピンボール風、ボールが上から落下、複数バーで跳ね返し、床の色/数字で結果分類 |
| 終了条件 | 全てのバーが削除される |
| スコア | 床着地時、数字（1-9）の床で得点、次回以降その床は+1ずつ増加 |
| 難易度 | 時間経過でボール落下速度が増加（difficulty * 0.5） |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| ball | char("a") | 上から落下（y += 0.5d）、バーで跳ね返され新座標に移動 | - |
| bar | bar(width, 3, angle) 白 | ランダム配置（7本）、角度±反転、タップで全バー角度符号反転 | ボール接触で跳ね返し（angle方向に移動） |
| floor (score) | box 薄黒/黒 | 下部に配置、数字1-9表示、着地で得点、次回+1増加 | ボール着地で得点＋消滅 |
| floor (+bars) | box 青 | +1～+4表示、ボール着地で表示数分バー追加 | 着地でバー追加＋消滅 |
| floor (-bars) | box 赤 | -1～-4表示、ボール着地でランダムに表示数分バー削除 | 着地でバー削除＋消滅 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | バーの角度反転、ボールの物理軌道、床の色分け（黒＝得点、青＝バー追加、赤＝バー削除）が一目瞭然 |
| (2) 視覚的フィードバック | バー角度のリアルタイム反転、床着地時のパーティクル、スコア数字の増加、バー追加/削除の視覚的変化 |
| (3) リスク・リワード | 赤床回避でバー保持、青床狙いでバー増加→操作余地拡大、高スコア床は次回さらに高得点化、選択が累積 |
| (4) 斬新なメカニクス | 全バー同時角度反転、床の永続的スコア増加システム、バー数動的変化でゲーム性自体が変動 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #rule-classify | 着地した床の種類で結果分類（得点/バー追加/バー削除）、色と数字で視覚化 | ボール軌道予測で着地先選択、高得点 vs バー維持 vs バー増加の戦略判断、習熟で狙い撃ち |
| #on_pressed-reverse_state | タップで全バーの角度符号を反転（barAngleSign *= -1）、ボール軌道を大きく変更 | タイミング一発で全体状態変化、ボール位置と目標床への軌道計算、失敗でバー全損リスク |
| #rule-physics | ボールがバーに当たると角度方向に跳ね返される物理挙動、慣性と反射 | 軌道予測の習熟曲線、バー配置と角度の複合計算、物理感覚が得点に直結 |
| #field-pins | 複数バー（7本初期配置、角度±反転）がピン状に配置、ボール軌道を複雑化 | バー数が多いほど制御性向上、削除で難化、配置ランダム性で毎回異なる戦略要求 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | バー反転をランダム個別反転に変更、タップで1本だけ反転、どれが反転するか予測不能で緊張感 |
| Combine (結合) | 床にタイマー追加、一定時間内に着地しないと高得点床が低得点化、スピードとリスクのトレードオフ |
| Modify (修正) | バー角度を長押し時間で微調整可能に、タップ＝反転、長押し＝角度調整で精密制御とリスク管理 |
