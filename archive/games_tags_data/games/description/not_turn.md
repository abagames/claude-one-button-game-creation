# NOT TURN (not_turn)

**タグ**: #on_pressed-select_route, #on_holding-move, #field-lanes, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（lane切り替え）/ 長押し（スピードアップ） |
| 挙動 | 5本のlane上を自動で上昇、横棒がある地点でボタンを押すと隣のlaneへ移動、長押しで速度4倍 |
| 終了条件 | 赤い敵に衝突 |
| スコア | 敵が残した黄色ドットを取得、取得ごとにmultiplierが1増加 |
| 難易度 | difficultyの平方根に応じて全体の移動速度が上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char(a-d) | 5本のlane上を上昇、横棒で隣のlaneへ移動可能、長押しで速度4倍 | - |
| bars | box(light_blue) | ランダムな高さに配置される横棒、lane間の通路 | - |
| enemies | char(e, red) | 自動でlane上を下降し、横棒で自動的にlaneを切り替え、移動軌跡に黄色ドット生成 | 終了 |
| dots | char(i, yellow) | 敵の軌跡に6単位ごとに生成 | multiplier点獲得、multiplier+1 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 5本の縦線とそれをつなぐ横棒という幾何学的構成。色分け（プレイヤー黒、敵赤、ドット黄、横棒青）で役割が一目瞭然 |
| (2) 視覚的フィードバック | 敵との衝突は赤色で危険を示唆、ドット取得は黄色で報酬を視覚化、multiplier表示で連鎖ボーナスを明示 |
| (3) リスク・リワード | 長押しで速度4倍になり多くのドットを取得可能だが、敵との遭遇リスクも増加し操作精度が要求される |
| (4) 斬新なメカニクス | "NOT TURN"という逆説的タイトル通り、プレイヤーは自発的に曲がれず横棒の配置に依存する受動的な経路選択が新鮮 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #on_pressed-select_route | 横棒がある地点でボタンを押すことで左右のlaneを選択 | 敵の配置を予測し、ドットを最大化する経路を瞬時に判断する必要がある |
| #on_holding-move | 長押しで速度が4倍に上昇 | リスクを取って高速移動し多くのドットを回収するか、安全に低速で進むかの選択 |
| #field-lanes | 5本の平行なlaneで構成される縦スクロールフィールド | lane間の移動タイミングが戦略の核となる |
| #field-auto_scroll | プレイヤーの高さに応じて自動スクロール | 一定の緊張感を維持しつつ、プレイヤーは経路選択に集中できる |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 横棒を「ジャンプ台」に変え、縦方向の移動も選択可能にする |
| Combine (結合) | アイテム取得要素を追加し、一時的に横棒なしでもlane移動可能にする |
| Modify (修正) | 敵が残すドットの密度を敵の種類によって変化させ、リスク・リターンの多様化を図る |
