# THROW M (throw_m)

**タグ**: #weapon-artillery, #player-reciprocate, #on_holding-adjust_angle, #on_released-throw

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押しで角度調整（左回転）、離すと投擲 |
| 挙動 | 右端で上下往復移動、砲弾を放物線で発射、敵と弾を破壊してマルチプライヤー獲得 |
| 終了条件 | 敵の弾がプレイヤーに接触 |
| スコア | 敵撃破でマルチプライヤースコア（倍々増加）、敵弾破壊で+1、投擲でリセット |
| 難易度 | 時間経過で敵の出現頻度・移動速度、弾速、角度回転速度、プレイヤー移動速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("b","c") 黒/青 | 右端を上下往復（19-90, vy *= -1）、自動移動 | 敵弾接触でゲームオーバー |
| shot | char("e") 黒 | 長押しで角度回転、離すと発射、放物線軌道（重力 += 0.07d） | - |
| enemy | char("b","c") 色付き | 上から降下（vy: 0.1-0.4d）、定期発射（200-300/√d間隔） | ショット命中で落下状態、マルチプライヤー×2 |
| bullet | char("d") 黒 | 敵から右方向へ水平移動（bs = 0.5d） | ショット破壊：得点＋マルチプライヤー+1、プレイヤー：死亡 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 角度ラインの視覚化、往復移動のリズム、放物線軌道の物理感、敵の色分けが明確 |
| (2) 視覚的フィードバック | 角度調整ラインのリアルタイム表示、敵撃破パーティクル、マルチプライヤー表示、弾破壊の視覚効果 |
| (3) リスク・リワード | 長押しで精密狙い vs 角度オーバーで自動発射、マルチプライヤー維持 vs 投擲タイミング、敵弾破壊で+1 |
| (4) 斬新なメカニクス | 往復移動と角度調整の同時管理、マルチプライヤー倍々増加（×2）vs 線形増加（+1）の使い分け |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #weapon-artillery | 放物線砲撃（angle初期135度、重力あり）、角度で軌道調整、リード射撃必須 | 移動予測と弾道計算、高角度で遠距離、低角度で速達、習熟で狙撃精度向上 |
| #player-reciprocate | 右端を自動上下往復（19-90, vy *= -1, 速度0.5d）、位置制御不可 | 発射タイミングが位置依存、リズム感覚が重要、往復周期と敵配置の読み合い |
| #on_holding-adjust_angle | 長押しで角度左回転（+= 0.1d）、135度から247.5度まで、超過で自動発射 | 精密照準 vs タイムリミット、角度範囲制約で戦略的選択、難易度上昇で回転加速 |
| #on_released-throw | 離した瞬間に砲弾発射（vel = 3√d）、マルチプライヤー1にリセット | 最適タイミングでマルチプライヤー最大化、我慢 vs 即発射、リセットペナルティで慎重さ要求 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 往復移動を円周移動に変更、画面中央を周回しながら全方向に砲撃、360度の角度管理 |
| Combine (結合) | 敵の色に応じて弾の属性変化、赤＝爆発範囲攻撃、青＝貫通弾、黄＝追尾弾、色選択で戦術変化 |
| Modify (修正) | 長押し時間で弾速・威力変化、短押し＝速射低威力、長押し＝強力遅延、チャージ管理とリスク |
