# GROWTH (growth)

**タグ**: on_holding-extend, field-3D, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し（成長） |
| 挙動 | 長押しでサイズが大きくなり、離すと小さくなる。大きいほど遅く、小さいほど速い。自分より小さい敵を食べる |
| 終了条件 | 自分より大きい敵に接触、または画面左（x < 0）に落ちる |
| スコア | 敵を食べると (速度 × 敵のサイズ) 点 |
| 難易度 | 時間経過で速度倍率が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | rect(size, size) yellow | 長押しでサイズ拡大（最大50）、離すと縮小（最小5）。サイズに反比例して速度変化 | - |
| floor | rect(200, 10) light_blue | 地面 | - |
| enemies（小） | rect(3, 3) cyan | 右から左へスクロール。遠くにあると3D的に縮小表示 | プレイヤー > 敵: 食べてスコア獲得 |
| enemies（大） | rect(size, size) red | 右から左へスクロール。サイズ7〜最大。遠くにあると3D的に縮小表示 | プレイヤー < 敵: ゲームオーバー |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 長押しで「大きく」なる直感的操作。色（cyan/red）で食べられる/食べられないが明確 |
| (2) 視覚的フィードバック | サイズの変化、速度の変化、3D的な遠近感（敵の縮小表示）。食べた時のパーティクルとスコア表示 |
| (3) リスク・リワード | 小さくすると速くて敵を避けやすいが、画面外に落ちるリスク。大きくすると安定だが遅く、大きい敵に追いつかれやすい |
| (4) 斬新なメカニクス | サイズと速度の反比例関係。3D的な遠近感で敵が徐々に大きくなる演出。動的なサイズ調整による戦略的深さ |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| on_holding-extend | 長押しでサイズが拡大（最大50）、離すと縮小（最小5） | サイズ調整が戦略の核心。敵に応じて最適なサイズを選ぶ習熟要素 |
| field-3D | 遠くの敵が小さく表示され、近づくと大きくなる3D的演出 | 敵の接近を視覚的に予測できる。奥行き感がゲームに深みを与える |
| field-auto_scroll | プレイヤーの速度と位置に応じてスクロール | 前進し続けないと画面外に落ちる。速度管理が重要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | サイズを「質量」に変更。大きいほど慣性が大きく、小さいほど機敏に動ける |
| Combine (結合) | 食べた敵の数に応じて「レベルアップ」し、特殊能力（ダッシュ、ジャンプ）を獲得 |
| Modify (修正) | サイズに上限と下限を設け、「エネルギー」として管理。長押しでエネルギー消費、敵を食べて回復 |
