# LINE B (line_b)

**タグ**: #player-reflect, #on_pressed-attack, #field-outpost

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | 2つの点を結ぶ線が壁で反射しながら移動、Tapで線が縮み、拡大する金色の円を捕獲 |
| 終了条件 | 金色の円が壁に接触するとゲームオーバー |
| スコア | 円を線で捕獲すると得点（円の半径に応じて増加: radius * sqrt(radius)） |
| 難易度 | 時間経過で円の拡大速度と線の移動速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| line | line(20px) | 2点が壁で反射、Tap時は中心に向かって加速し線が縮む | - |
| gold | arc(radius) | 徐々に拡大（0.05 * difficulty/tick） | 線で捕獲→得点、壁接触→終了 |
| walls | rect(4辺) | 画面境界（light_purple） | 線を反射、円接触で終了 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 画面内を移動する1本の線と、拡大する円という極めてシンプルな構成。線の軌跡が薄く表示され、動きが予測しやすい |
| (2) 視覚的フィードバック | 壁に当たると「hit」音、円捕獲で「powerUp」音。線の軌跡履歴が残り、動きのパターンが視覚化される。円が壁に近づく危険性が色で明示 |
| (3) リスク・リワード | 円が大きくなるほど高得点だが壁に接触するリスクも増大。Tapで線を縮めるタイミングが遅いと高得点、早いと安全だが低得点 |
| (4) 斬新なメカニクス | プレイヤーが「点」ではなく「線」という2次元的な存在。線の両端が独立して壁で反射するため、複雑な軌道を描き予測困難な動きが生まれる |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-reflect | 線の両端が壁で独立に反射し、絶えず形状と角度が変化 | 反射のパターンを読み、円が出現する安全な位置を予測する習熟要素 |
| #on_pressed-attack | Tapで線の両端を中心方向に加速させ、線を縮めて円を捕獲 | タイミングが全て。円が大きくなるのを待つか、早めに確実に取るかの判断 |
| #field-outpost | 画面4辺が壁であり、同時に円の死のライン | 壁際での円の出現はハイリスク。線の軌道を壁から遠ざけるか、素早く対処するか |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 線を弾性のあるゴムやバネに変更し、伸縮に物理的な反動を追加 |
| Combine (結合) | 複数の円が同時に出現し、全てを線で囲むボーナスや、円同士が接触すると爆発するギミック |
| Modify (修正) | 線の端点を3つ以上に増やし、三角形や四角形で円を捕獲。または線の長さに制限を設け、伸びすぎると切れる |
