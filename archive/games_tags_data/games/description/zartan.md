# ZARTAN (zartan)

**タグ**: #player-string, #field-pins, #on_pressed-jump_into_another, #field-bottomless, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す / 長押し |
| 挙動 | 重力で落下するプレイヤーを操作し、ボタンで最も近い杭にロープを接続、長押しで引っ張られてスイング、離すと解放 |
| 終了条件 | プレイヤーが画面下端（y=99）に到達 |
| スコア | 画面左側を維持する（x>30でボーナス）＋時間経過 |
| 難易度 | 時間経過でスクロール速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | box(7,7) | 重力で落下、慣性で移動、天井で跳ね返る | - |
| pile | box(5,5) | 画面右から出現し左へスクロール | - |
| nearest | box(9,9) cyan | 最も近い杭をハイライト表示 | ボタンでアンカーに設定 |
| rope | line | アンカーとプレイヤーを結ぶロープ、長押しで引っ張る力が発生 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 最も近い杭がシアン色でハイライトされ、ターゲットが明確。ロープの線で接続状態が視覚的に分かる。緑のプレイヤーと黒い杭のコントラストが明確 |
| (2) 視覚的フィードバック | ロープ接続時の音とライン表示で状態が明確。慣性の動きで物理的な手応えがある。画面左側維持でスコアが増加する仕組みが直感的 |
| (3) リスク・リワード | 高い位置の杭を選ぶと安全だが移動距離が短く、低い位置の杭は大きくスイングできるがリスクが高い。ロープを早く離すと勢いがつく |
| (4) 斬新なメカニクス | 「最も近い杭を自動選択」システムがユニーク。プレイヤーは位置取りとタイミングに集中でき、複雑な選択操作が不要。慣性とロープの物理的なスイング感が爽快 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-string | ロープでアンカーに接続し引っ張られる | スイングの軌道を予測し、次の杭へのタイミングを計る。ロープの長さが振り子の動きを決定 |
| #field-pins | 杭が不規則な高さで配置、スクロールで流れる | 杭の配置パターンを先読みし、最適なルートを選択。高低差が戦略の幅を生む |
| #on_pressed-jump_into_another | ボタンで最も近い杭にロープ接続 | 位置取りで次のターゲットを制御。プレイヤーの高さと水平位置が選択される杭を決定 |
| #field-bottomless | 画面下端に落ちるとゲームオーバー | 高度管理が最優先。下降しすぎる前にロープで引き上げる必要がある |
| #field-auto_scroll | 左方向への自動スクロール、速度は時間で増加 | 画面左側を維持しながら前進する必要。スクロール速度に合わせたスイングのリズムが重要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 杭を「移動するプラットフォーム」に置き換え、動的な目標への接続を要求 |
| Combine (結合) | 複数の杭に同時接続できるようにし、2本のロープでの複雑なスイングを実現 |
| Modify (修正) | ロープの長さを長押し時間で調整可能にし、スイングの軌道をより細かく制御 |
