# REFBALS (refbals)

**タグ**: #player-multiple, #player-reflect, #field-floors, #field-bottomless

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押しで加速 |
| 挙動 | 複数のボールが重力で落下し、床で反射して跳ね返る |
| 終了条件 | ボールが画面下端に到達 |
| スコア | ボールが床に反射するたびに +1点 |
| 難易度 | なし（スコアベース） |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| ball | box(5, 5) | 重力で落下（v += 0.03） | 床で反射、下端で終了 |
| wall | box(36, 3) | 長押しで速く（2）、離すと遅く（1）横スクロール | ボールを反射→+1点 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 紫のボールと青い床というミニマルな構成。重力と反射の物理法則が直感的に理解できる。 |
| (2) 視覚的フィードバック | ボールの落下と跳ね返りのアニメーション、床のスクロール速度の変化が明確。 |
| (3) リスク・リワード | 床を速く動かすとボールの下に配置しやすいが、予測が難しくなる。複数のボールを同時に管理する技術。 |
| (4) 斬新なメカニクス | 複数のボールを同時に管理し、床の速度を調整して全てのボールを跳ね返し続ける独特のバランス感覚。 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-multiple | 複数のボールが同時に存在し管理が必要 | 複数の要素を同時に追跡し、優先順位をつける戦略性を生む |
| #player-reflect | ボールが床で反射して跳ね返る | 物理法則を利用した予測可能な動きにより、習熟によるコントロールを実現 |
| #field-floors | 横にスクロールする床 | ボールの着地位置を調整するための操作対象を提供 |
| #field-bottomless | 下端に到達するとゲームオーバー | 継続的なプレッシャーと失敗条件を生み出す |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ボールをキャラクターに、床をトランポリンに変更してテーマを変える |
| Combine (結合) | 複数の床を異なる速度で動かし、より複雑な管理を要求 |
| Modify (修正) | 長押し時間でボールの反射力が変化し、高く跳ね上がる |
