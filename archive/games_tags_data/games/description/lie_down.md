# LIE DOWN (lie_down)

**タグ**: on_holding-avoid, field-holes, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | 長押しで伏せて岩と穴を回避、離すと自動加速 |
| 終了条件 | 岩に当たる、穴に落ちる、または画面左端到達 |
| スコア | 敵を避けると得点、連続回避で倍率上昇（長押しで倍率減少） |
| 難易度 | 時間経過で敵の出現頻度と速度、穴の出現頻度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("a"/"b"/"c") | 自動加速、長押しで伏せる（高さ変化） | - |
| enemy | char("d"/"e") | 上から降下、着地時に岩を投擲 | 回避で+得点 |
| rock | char("f") 赤 | 敵が投げる、右方向に加速 | 接触で終了 |
| hole | 白い矩形 | 地面に出現、上下に伸縮 | 穴の間隔が狭いと落下終了 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 長押しで伏せる直感的操作、敵と岩の視覚的区別 |
| (2) 視覚的フィードバック | 敵の降下軌道、岩の投擲、穴の伸縮アニメーション |
| (3) リスク・リワード | 敵回避で高得点だが長押しで倍率減少、伏せタイミングの判断 |
| (4) 斬新なメカニクス | 長押しでの姿勢変化、穴と岩の複合的な回避判断 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| on_holding-avoid | 長押しで伏せて岩を回避 | 岩と敵のタイミングを見極め、最小限の長押しで回避 |
| field-holes | 地面に伸縮する穴 | 穴の位置と幅を予測、伏せて穴を回避するか加速で飛び越えるか判断 |
| field-auto_scroll | プレイヤー位置に応じてスクロール | 前進維持の緊張感、加速と減速のバランス調整 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 伏せをジャンプに変え、空中の障害物を避けるゲームに |
| Combine (結合) | 長押し時間で伏せの深さを変え、複数段階の回避高さを実装 |
| Modify (修正) | 敵の投擲物を複数種類に分け、色に応じた回避方法を追加 |
