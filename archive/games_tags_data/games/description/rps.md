# RPS (rps)

**タグ**: #field-lanes, #rule-match, #weapon-blow_off, #on_pressed-reverse_state, #field-outpost

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | タップで右のレーンに移動し、じゃんけんの手を変更（rock→paper→scissors→rock）。上から降ってくる手と対戦し、勝てば破壊、引き分けは押し返し、負けると吹き飛ばされる。 |
| 終了条件 | 手が画面下端に到達 |
| スコア | 勝利で得点、連勝でmultiplier増加。負けるとmultiplierリセット |
| 難易度 | 時間経過で手の降下速度と出現頻度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| myHand | char(a/b/c) cyan/purple/yellow | タップでレーン移動 & 手変更、自動的に上昇 | - |
| hands | char(a/b/c) cyan/purple/yellow | 上から降下、互いに押し合う物理挙動 | 勝ち: 破壊 / 引き分け: 押し返し / 負け: プレイヤー吹き飛ばし |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | じゃんけんという普遍的ルール、色分け（cyan=rock, purple=paper, yellow=scissors）で視認性確保。4レーンの固定配置が明快。 |
| (2) 視覚的フィードバック | 勝利時の破壊エフェクト、引き分け時の押し合い、負け時の吹き飛ばしが明確。手同士の物理的な押し合いが視覚的。 |
| (3) リスク・リワード | 連勝でmultiplier上昇だが、負けると全リセット。レーン選択とタイミングがスキルを要求。複数の手が重なる状況での判断が重要。 |
| (4) 斬新なメカニクス | じゃんけんを縦スクロールゲームに落とし込み。タップ1回でレーン移動と手変更を同時実行。手同士の物理的な押し合いが独特。 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-lanes | 4つの固定レーンを移動 | レーン選択が戦略の核心。手の配置を見極め、最適なレーンに移動 |
| #rule-match | じゃんけんのマッチングルール（勝ち/引き分け/負け） | 手の種類を予測し、適切な手で対応。3すくみの関係が戦略性を生む |
| #weapon-blow_off | 負けるとプレイヤーが下方向に吹き飛ばされる | 負けのペナルティが大きく、慎重な判断が必要。画面下端到達のリスク |
| #on_pressed-reverse_state | タップで手の種類を循環的に変更 | 1回のタップで移動と手変更が同時。次の手を予測した事前準備が重要 |
| #field-outpost | 4つの固定レーン、各レーンで独立した手が出現 | 全レーンの状況を把握し、優先順位を判断する全体視野が求められる |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | じゃんけんを色マッチングに変更し、同じ色で消去 |
| Combine (結合) | 複数の手を同時に出せるようにし、2対1の対戦を可能に |
| Modify (修正) | 手の種類を4つ以上に増やし、複雑なマッチングルールを追加 |
