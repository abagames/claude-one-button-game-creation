# TWO LANE (two_lane)

**タグ**: #field-lanes, #on_pressed-reverse_state, #on_holding-thrust

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップ：レーン切り替え / 長押し：加速 |
| 挙動 | 左右2つのレーンを走る車を操作。タップでレーン切り替え、長押しで加速。コインを集めてスコアとマルチプライヤーを獲得 |
| 終了条件 | コースアウト（道路の外）して速度が0.1以下になる |
| スコア | コイン取得時に (速度²+1) × マルチプライヤー のスコア獲得 |
| 難易度 | 時間経過で基本速度と最大速度が上昇。道路が片側のみになる頻度も増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| car(a) | char("a") 黒色 | 左右レーンを移動。タップで反対側に切り替え。速度に応じてスムーズに移動 | - |
| road | rect 薄黒/白 | 上にスクロール。両側、左のみ、右のみの3パターン。白線で視覚的に区別 | 道路上にいないとコースアウト |
| coin(b) | char("b") 黒色 | 左または右レーン上にランダム出現。20%の確率で生成 | (速度²+1) × マルチプライヤー のスコア + マルチプライヤー+1 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 2つのレーンと車、黒いコインという最小限の要素。レーシングゲームの直感的な操作感 |
| (2) 視覚的フィードバック | コイン取得で「coin」音と「+スコア×マルチプライヤー」表示。コースアウトで「hit」音と減速。速度の変化が視覚的に明確 |
| (3) リスク・リワード | 加速するほど高スコアだが、レーン切り替えの難易度が上昇。コインを逃すとマルチプライヤー減少という選択的ペナルティ |
| (4) 斬新なメカニクス | 片側レーンの出現がレーン選択を強制。スピードがスコアに直結し、コースアウトで即終了ではなく減速という猶予 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-lanes | 左右2つのレーンに分かれた道路。時々片側のみになる | レーン選択の先読みが重要。片側レーンを予測し、安全なレーンにいるか判断が生存を左右 |
| #on_pressed-reverse_state | タップで左右レーンを切り替え | レーン切り替えのタイミングが攻略の核心。コイン取得と片側レーン回避の両立が求められる |
| #on_holding-thrust | 長押しで速度が上昇し、離すと基本速度に戻る | 高速でスコアを稼ぐか、低速で安全にプレイするかの選択。マルチプライヤーとの兼ね合い |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 2レーンを3レーン、4レーンに増やし、より複雑な選択を要求。または縦スクロールを横スクロールに変更 |
| Combine (結合) | 障害物を追加。道路上に車や障害物を配置し、レーン切り替えと障害物回避の両立を要求 |
| Modify (修正) | 道路の幅をランダムに変動。広いレーンと狭いレーンが混在し、タイミング調整の難易度を上げる |
