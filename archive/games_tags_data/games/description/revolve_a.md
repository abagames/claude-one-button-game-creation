# REVOLVE A (revolve_a)

**タグ**: #on_pressed-go_forward, #field-connected, #rule-chain, #field-pins, #player-rotate

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップで前進 |
| 挙動 | 回転する矢印が壁で反射し、緑のラインで繋がった敵に触れると連鎖破壊 |
| 終了条件 | 矢印が敵に直接衝突 |
| スコア | 連鎖破壊した敵の数 × multiplier（連鎖数で増加） |
| 難易度 | 矢印の回転速度と敵の出現頻度・移動速度が上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| arrow | line（十字） | 自動で回転、タップで前進、壁で反射 | 敵で終了、緑線で連鎖破壊 |
| enemy | char("a") | ランダムな方向に移動 | 直接衝突で終了、連鎖で破壊 |
| line | line（緑） | 距離30以内の敵同士を結ぶ | 矢印が触れると連鎖破壊 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 回転する矢印、敵、それらを結ぶ緑のラインという明快な構成。タップで前進という直感的な操作。 |
| (2) 視覚的フィードバック | 矢印の回転と前進、敵同士を結ぶ緑のラインの動的な変化、連鎖破壊のパーティクル効果。 |
| (3) リスク・リワード | 緑のラインを狙うと連鎖破壊で高得点だが、敵に近づくリスク増。タップのタイミングと方向の判断。 |
| (4) 斬新なメカニクス | 敵同士を結ぶラインを切断することで連鎖破壊する独特のシステム。回転する矢印による予測不可能な軌道。 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #on_pressed-go_forward | タップで矢印が現在の角度で前進 | 回転のタイミングを見極めて最適な方向に前進する戦略性を生む |
| #field-connected | 近い敵同士が緑のラインで結ばれる | 敵の配置を視覚化し、連鎖破壊の機会を明確にする |
| #rule-chain | ラインに触れた敵から再帰的に連鎖破壊 | 一度のアクションで複数の敵を破壊する戦略性と爽快感を提供 |
| #field-pins | 敵がピンのように配置され、ラインで結ばれる | 空間的な配置と連鎖の可能性を視覚化 |
| #player-rotate | 矢印が自動で回転し続ける | 継続的な変化により、タイミングの判断を要求 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 矢印を剣や光線に、ラインを鎖や磁力線に変更してテーマを変える |
| Combine (結合) | 複数の矢印を同時に操作し、異なる方向から連鎖を狙う |
| Modify (修正) | 長押しで回転速度を調整可能にし、より精密な狙いを実現 |
