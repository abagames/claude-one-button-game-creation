# ZONE B (zone_b)

**タグ**: #player-limited_vision, #on_holding-shoot, #on_pressed-turn

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す / 長押し |
| 挙動 | 4方向に自動移動するプレイヤーを操作。長押しで停止して射撃、ボタンを離すと90度回転して再び移動開始 |
| 終了条件 | 敵弾に当たる、青いゾーン外に出る、またはゾーンが消滅 |
| スコア | 敵を倒すと得点（連続撃破でコンボ倍率増加） |
| 難易度 | レベルが上がると敵の数が増加（9+レベル×7体） |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char(a/b) blue | 4方向に移動、長押しで停止＋射撃、離すと回転、壁で反射 | - |
| enemy | char(a/b) red | 4方向に移動、定期的にバースト射撃（3-7発）、壁で反射、視界内のみ表示 | プレイヤー弾で撃破＋得点 |
| wall | box yellow | 固定配置、ゾーン縮小で消滅 | 反射 |
| bullet | char(c/d) purple/cyan | 4方向に直進、射程20、視界外の敵弾は透明 | 壁または範囲外で消滅 |
| zone | arc blue | 時間経過で縮小・移動、予告表示あり | ゾーン外でゲームオーバー |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 4方向移動とシンプルな回転操作。視界範囲（30px）外は透明で、周囲の状況に集中できる。青いゾーンで安全エリアが明確 |
| (2) 視覚的フィードバック | 視界内の敵のみ表示され、周囲の脅威が視覚的に限定される。ゾーン縮小の予告（薄い円）で危険が事前に伝わる。射撃時の停止で攻守のトレードオフが明確 |
| (3) リスク・リワード | 射撃には停止が必要で無防備になる。視界制限により遠くの敵が見えず、探索にリスクがある。ゾーン縮小により戦闘エリアが狭まり、敵との遭遇率が上昇 |
| (4) 斬新なメカニクス | 視界制限が緊張感を生み、バトルロワイヤル風の縮小ゾーンが時間制限として機能。移動と射撃の二択が戦術的判断を要求 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-limited_vision | 周囲30ピクセル内の敵と弾のみ表示 | 不意打ちのリスクがあり、常に周囲を警戒する必要。視界外から飛来する弾は見えないため、音と状況判断が重要 |
| #on_holding-shoot | 長押しで停止して射撃（7フレーム間隔） | 移動と射撃の二択。射撃中は無防備なため、敵の位置と弾道を予測して安全な位置で射撃するタイミングが重要 |
| #on_pressed-turn | ボタンを離すと90度回転（移動中のみ） | 壁や敵の位置に応じた素早い方向転換が可能。離すタイミングで進行方向を制御し、戦術的な位置取りを実現 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 視界制限を「霧」や「暗闇」の視覚効果に置き換え、雰囲気を強化 |
| Combine (結合) | 「アイテム収集で視界範囲拡大」要素を追加し、探索の報酬を導入 |
| Modify (修正) | 移動中も射撃可能にするが精度が低下、停止射撃は高精度という選択肢を追加 |
