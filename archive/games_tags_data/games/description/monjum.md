# MONJUM (monjum)

**タグ**: #on_holding-stop, #on_released-throw, #field-holes

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し（後退）/ 離す（ジャンプ） |
| 挙動 | 穴あきの床を走り、Holdで後退しタイミングを計り、Releaseでジャンプ。空中で敵・穴・コインを通過し、着地時に組み合わせで得点 |
| 終了条件 | 穴に落ちる、または敵に接触するとゲームオーバー |
| スコア | 着地時にビットカウントで得点（コインのみ=1点、コイン+穴=5点、コイン+穴+敵=20点） |
| 難易度 | スクロール速度、ジャンプ力、敵の出現頻度が時間経過で増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char(a/b) | Hold時に後退、Release時にジャンプ、通常は画面中央に自動復帰 | 穴に落ちる→終了 |
| floor | rect(幅50-90) | 自動スクロール、間に穴が開く | - |
| enemy | char(c/d) | 左に移動、穴の前で自動ジャンプ | 接触→終了、ジャンプ中に下を通過→得点要素 |
| coin | text($) | 空中に浮遊、自動スクロール | 取得→得点要素 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | Hold=後退、Release=ジャンプという単純な操作。穴（床の切れ目）と敵（紫）、コイン（黄色）が明確に区別される |
| (2) 視覚的フィードバック | ジャンプ時に「jump」音、着地時に「coin」音と得点表示。敵接触/穴落下で「explosion」音。得点が組み合わせ数で変化し、達成感を演出 |
| (3) リスク・リワード | 高得点を狙うには敵・穴・コインの3要素すべてを1回のジャンプで通過する必要があり、タイミングが極めて重要。安全なジャンプは低得点 |
| (4) 斬新なメカニクス | 「ジャンプ中に何を通過したか」がビットフラグで記録され、組み合わせで得点が決まる独特のシステム。チャージ時間ではなく、タイミングと配置の読みが重要 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #on_holding-stop | Hold時にプレイヤーが後退し、自動前進を停止 | ジャンプのタイミングを調整する手段。敵・穴・コインの配置を見極め、最適な位置からジャンプする準備 |
| #on_released-throw | Release時にジャンプ（投げる動作のように弾道を描く） | チャージではなくタイミング。後退距離よりも、いつReleaseするかの判断が重要 |
| #field-holes | 床に穴があり、落ちると即死 | 穴は危険だが、高得点の必須要素。穴の上を通過するリスクを取るか、安全に避けるか |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 後退を「チャージ」に変更し、Hold時間に応じてジャンプ距離が変化する仕組みに |
| Combine (結合) | アイテムの種類を増やし、より多様な組み合わせ（4種で50点など）を追加 |
| Modify (修正) | 着地時だけでなくジャンプ中にもリアルタイムで得点表示し、どの要素を通過したかをフィードバック |
