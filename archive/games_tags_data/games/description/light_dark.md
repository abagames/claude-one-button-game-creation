# LIGHT DARK (light_dark)

**タグ**: #player-inverted, #on_pressed-reverse_state, #on_pressed-jump, #field-spike, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（地上）/ 押す（空中）/ 長押し |
| 挙動 | 画面上下に分かれた明と暗の世界を切り替えながら、各世界のコインを回収しスパイクを回避 |
| 終了条件 | 自分の世界のスパイクに接触するとゲームオーバー |
| スコア | コイン回収で連続multiplier点（1, 2, 3...）、取り逃すとmultiplier減少 |
| 難易度 | スクロール速度が時間経過で増加、ジャンプ力も難易度に応じて上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char(a/b) | 地上でTap→世界反転＋ジャンプ、空中でTap→世界反転、Hold→落下減速 | - |
| coin | text(o) | 自動スクロール、lightとdarkの2レーン | +multiplier点、連鎖ボーナス |
| spike | char(c) | 自動スクロール、コインの波の後に出現 | 終了 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 画面を上下に分割し、明暗を色分け（黒/白の背景反転）。プレイヤーキャラクターも世界に応じて色が反転し、どちらの世界にいるかが一目瞭然 |
| (2) 視覚的フィードバック | Tap時に「jump」「laser」の効果音で世界切り替えを明示。コイン回収時に得点表示＋multiplier表示で連続ボーナスを可視化。スパイク衝突で「explosion」音 |
| (3) リスク・リワード | 両世界のコインを効率よく回収するには頻繁な世界切り替えが必要だが、切り替え直後はスパイクのタイミングが見えにくくリスクが高い。multiplier維持のためコインを逃せない緊張感 |
| (4) 斬新なメカニクス | 1つのゲーム空間に2つの世界が重なり、ボタン1つで瞬時に世界を切り替える構造。各世界は独立したスクロール・障害物配置を持ち、2つのゲームを同時にプレイする感覚 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-inverted | プレイヤーが画面上下で反転した2つの世界を行き来 | 同一画面内の2つの視点を素早く切り替える認知能力が求められる |
| #on_pressed-reverse_state | ボタン押下で明暗の世界を反転＋ジャンプ（地上時）または反転のみ（空中時） | 1つの入力が状況に応じて異なる効果を生み、戦略的な使い分けが生まれる |
| #on_pressed-jump | 地上にいるときのボタン押下でジャンプ | ジャンプのタイミングと世界切り替えのタイミングを同時に判断する複雑性 |
| #field-spike | 画面に固定スパイクではなく、スクロールするスパイクの波 | 両世界のスパイク配置を記憶・予測し、安全な切り替えタイミングを見極める必要 |
| #field-auto_scroll | 自動スクロールで障害物が接近 | 反応速度が直接スコアに影響、習熟で先読みと最適ルート選択が可能に |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 明暗の2世界を「重力の向き」に置き換え、上下反転のプラットフォーマーに変化 |
| Combine (結合) | 両世界で同時にアイテムを取ると特別ボーナス、または片方の世界の行動がもう一方に影響を与えるギミック追加 |
| Modify (修正) | 世界切り替えにクールダウンを設け、より計画的な切り替えを要求。または3つ以上の世界を循環させる |
