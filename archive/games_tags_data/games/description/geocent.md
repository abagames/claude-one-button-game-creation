# GEOCENT (geocent)

**タグ**: #field-space, #player-follow, #on_holding-move, #on_holding-rotate

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し（加速 & 回転速度増加） |
| 挙動 | ロケットが惑星の周りを公転。長押しで加速し内側に移動、離すと減速し外側に戻る。目標惑星に到達でクリア |
| 終了条件 | 惑星または太陽に接触 |
| スコア | クリア時の貨物（crates）数 × 1点 |
| 難易度 | 時間経過で惑星の公転速度増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| rocket | bar(1,3) + bar(2,3) blue | 惑星の周りを公転。長押しで内側に移動、回転速度上昇 | 惑星/太陽接触: ゲームオーバー |
| crates（貨物） | box(2) black | ロケットの後ろに連なる。プレイヤーの軌跡に沿って追従 | 惑星/太陽接触: 消滅 |
| stars（惑星） | arc(size) 5色 | 太陽の周りを公転。9つの惑星が異なる速度で回転 | - |
| sun（太陽） | arc(3) yellow | 中央の黄色い惑星。到達目標 | - |
| target（目標惑星） | arc(2) cyan | 青い目標惑星。ここに到達するとクリア | 到達: 貨物数に応じてスコア、次の周回へ |
| bg stars | box(1) light_black | 背景の星 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 長押しで「内側に入る」という直感的操作。惑星の軌道と色で視覚的に区別。目標惑星の青色が明確 |
| (2) 視覚的フィードバック | ロケットの軌跡パーティクル、貨物の追従、惑星の公転アニメーション。接触時の爆発 |
| (3) リスク・リワード | 内側に入るほど危険だが最短ルート。貨物が多いほど高得点だが、接触リスクで全損の可能性 |
| (4) 斬新なメカニクス | 公転軌道を動的に変える独自システム。貨物が軌跡に沿って追従し、Snake風の要素。長押し時間で距離と回転速度を同時制御 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| field-space | 宇宙空間で惑星が公転する3D的な環境 | 惑星の動きを予測し、タイミングを計る必要がある。視点が目標惑星に固定され、相対的な動きが重要 |
| player-follow | 貨物がロケットの軌跡に追従 | 貨物の安全確保が戦略的。長い貨物列は高得点だが、接触リスクも増大 |
| on_holding-move | 長押しで内側に移動（加速） | 長押し時間で軌道をコントロール。短いタップで微調整、長押しで大胆な移動 |
| on_holding-rotate | 長押しで回転速度増加 | 回転速度が上がることで、惑星を早く回り込める。タイミングとリスク管理が習熟要素 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 惑星を「障害物リング」に変更。リングの隙間を通過する |
| Combine (結合) | 燃料システムを追加。長押しで燃料消費、燃料切れで減速 |
| Modify (修正) | 貨物を「重力影響」に。貨物が多いほどロケットが重くなり、加速が鈍る |
