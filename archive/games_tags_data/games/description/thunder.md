# THUNDER (thunder)

**タグ**: #on_pressed-turn, #field-weather, #rule-proximity_bonus, #rule-combo_multiplier

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップで左右反転 |
| 挙動 | 地面を左右移動、雷が上から枝分かれしながら成長、発動前に星を回収してマルチプライヤー獲得 |
| 終了条件 | 発動中の雷（黄色）に接触 |
| スコア | 星回収でマルチプライヤースコア（連続回収で増加）、雷発動でリセット |
| 難易度 | 時間経過で雷の伸長速度・枝分かれ速度、プレイヤー移動速度、星の落下速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("b","c") 黒 | 地面を左右移動（vx: ±√d）、端到達またはタップで反転 | 黄色雷接触でゲームオーバー |
| lightning | line(2) 薄黒 → line(4) 黄 | 上から枝分かれ成長（1-3分岐）、地面到達で全体が黄色発動 | 発動前：無害、発動中：接触で死亡 |
| star | char("a") 黄 | 雷線の各端点から生成、落下＆地面バウンド（vy *= -0.5） | プレイヤー回収でマルチプライヤー得点 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 雷の枝分かれ成長、色変化（薄黒→黄）で危険明示、星の落下とバウンド、左右反転の単純操作 |
| (2) 視覚的フィードバック | 雷発動時のパーティクル爆発、星回収スコア表示、マルチプライヤー増加、枝分かれ音とリズム |
| (3) リスク・リワード | 雷近接で星密集、長時間待機で高マルチプライヤー vs 発動リスク、ギリギリ回収で高得点 |
| (4) 斬新なメカニクス | 枝分かれ成長パターン（1-3分岐、ランダム角度）、雷発動前の星生成システム、距離とリスクの相関 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #on_pressed-turn | タップまたは端到達で移動方向反転（vx *= -1）、即座に方向転換 | 雷の隙間への進入タイミング、星回収ルート最適化、反転リズムが習熟の核心 |
| #field-weather | 雷が上から枝分かれ成長（20-40tick後に1-3分岐）、160本超過または地面到達で全体発動 | 予測不能な成長パターン、発動タイミング読み、雷密度が戦場複雑化、毎回異なる配置 |
| #rule-proximity_bonus | 雷発動直前に星が各端点から生成、雷近くほど星密集、危険領域ほど高リターン | リスク取りで高マルチプライヤー、安全プレイ vs 攻撃的回収、位置取りで得点最大化 |
| #rule-combo_multiplier | 星回収で multiplier++、雷発動でリセット、連続回収で得点倍増 | 発動前の限られた時間で最大回収、発動タイミング予測、コンボ維持 vs 安全撤退 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 雷を味方エネルギーに変換、タップで枝の一つを消去、戦略的に安全地帯を作る vs 星源を減らす |
| Combine (結合) | 星の色を複数種類に分け、雷の枝ごとに色継承、同色連続回収でボーナス、色選択で戦略変化 |
| Modify (修正) | 長押しで時間減速（雷成長・星落下遅延）、チャージゲージ消費、精密回収 vs リソース管理 |
