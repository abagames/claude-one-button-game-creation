# PAINT BALL (paint_ball)

**タグ**: #player-paint, #player-rotate, #on_pressed-go_forward, #field-outpost, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（ボールを発射） |
| 挙動 | 左右の壁で回転する砲台から黄色ボールを発射し、12x15のグリッド上のマスを黄色に塗る。上から青いボールが落ちてきて青色に塗り替える |
| 終了条件 | 青いボールが画面下端に到達 |
| スコア | 青いマスを黄色に塗るとmultiplier点獲得、塗るごとにmultiplier+1。青いボールに直撃でもmultiplier点とmultiplier+1。発射するとmultiplierリセット |
| 難易度 | difficultyに応じてスクロール速度、ボール速度、青いボールの出現頻度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| waitingBall | char(a, yellow) + bar | 左右の壁で待機し、角度-45°～+45°で回転 | - |
| yellow_balls | char(a, yellow) | 発射されたボール、物理演算でバウンド、グリッドのマスを黄色に塗る | 壁で反射、黄色に塗ったマスを得点化 |
| blue_balls | char(b, blue) | 上から出現し落下、グリッドのマスを青色に塗る | 黄色ボールと衝突で消滅+得点 |
| grid | box(6px) | 12x15のグリッド、黄色/青/灰色の3色で塗り分けられる | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | グリッドと黄色/青の2色のボールというシンプルな構成、塗り替えの視覚的変化が明確 |
| (2) 視覚的フィードバック | 塗った色の変化がリアルタイムで反映、効果音でスコアとイベントを通知、multiplier表示で連鎖ボーナスを視覚化 |
| (3) リスク・リワード | 青いボールに直撃すれば即座に高得点だが、塗る機会を逃す。青いマスを塗れば確実に得点だが時間がかかる |
| (4) 斬新なメカニクス | 陣取りゲーム+回転砲台+auto-scrollの組み合わせ。青いボールが塗り替えてくる敵対的な要素が独特 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-paint | 黄色ボールでグリッドのマスを黄色に塗る | どのマスを優先的に塗るか、どの青いボールを狙うかの戦略的判断が求められる |
| #player-rotate | 砲台が左右の壁で角度-45°～+45°で自動回転 | 回転のタイミングを見計らって、目標のマスや青いボールを狙う必要がある |
| #on_pressed-go_forward | ボタンを押すと回転方向にボールを発射 | 発射タイミングが重要、角度と目標の位置を予測する必要がある |
| #field-outpost | 砲台が左右の壁に固定され、中央のグリッドを塗る | 砲台の位置が固定されているため、角度調整のみで対応する制約 |
| #field-auto_scroll | 下方向に自動スクロール、グリッドも一緒に移動 | 時間制限的なプレッシャー、スクロール速度を考慮した着弾点予測が必要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 青いボールを「消しゴム」に変え、黄色を消す要素に変化させる |
| Combine (結合) | 複数色のボールを導入し、色の組み合わせで高得点を狙う要素を追加 |
| Modify (修正) | グリッドを3Dの立方体に変え、複数の面を塗る立体的な陣取りゲームに拡張 |
