# FLIP O (flip_o)

**タグ**: player-multiple, rule-physics, on_pressed-reverse_state, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップでフリッパー反転 |
| 挙動 | 複数のボールが物理演算で跳ね回る。タップでフリッパーを反転してブロックを破壊 |
| 終了条件 | 全ボール落下、またはブロックが固定バンパーに接触 |
| スコア | ブロック破壊時にボール数×multiplier。赤ブロック破壊でボール追加 |
| 難易度 | 自動スクロール速度増加、タップ時の加速増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| ball | arc | 重力と慣性で移動。ボール同士も反射 | - |
| flipper (紫) | bar×2 | タップで0.5↔-0.5ラジアン反転 | タップ時にボールを弾く |
| block (水色/赤) | box | 上から自動スクロール | 破壊時に得点、赤はボール追加 |
| bumper (青) | bar×2 | 固定 | ボールを反射 |
| wall | rect | 画面端 | ボールを反射 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | ピンボールの直感的な物理挙動。タップでフリッパー反転という単純操作 |
| (2) 視覚的フィードバック | ブロックが上から迫る視覚的圧力。ボールの軌跡が物理的に予測可能。赤ブロックの特別な存在感 |
| (3) リスク・リワード | 赤ブロック破壊でボール増加だが、多ボールは制御困難。タップタイミングの精密さが得点とリスクを左右 |
| (4) 斬新なメカニクス | 自動スクロールピンボール。ブロックが降ってくる逆転構造。複数ボールの同時管理と物理的相互作用 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| player-multiple | 複数ボールの同時制御 | 1タップで全ボールに影響。ボール間の衝突も考慮した高度な予測が必要 |
| rule-physics | リアルな物理演算(重力・反射・摩擦) | 物理的直感が通用するゲームプレイ。習熟により軌道予測が可能に |
| on_pressed-reverse_state | フリッパーの状態を反転 | 2状態の切り替え。タイミングだけでなく、事前の状態認識も重要 |
| field-auto_scroll | ブロックが上から下へスクロール | 時間経過による圧力。ブロックがバンパーに到達する前に破壊が必要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | フリッパーを「重力反転ゾーン」に置き換え、タップでゾーン内の重力を反転してボールを弾く |
| Combine (結合) | rule:combo_multiplierを追加し、短時間内の連続破壊でmultiplier上昇。物理的な連鎖反応の報酬 |
| Modify (修正) | ボールにタイプ(重い/軽い/弾む)を追加し、それぞれ異なる物理特性。戦略的なボール選択 |
