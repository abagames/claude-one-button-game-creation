# SNAKY (snaky)

**タグ**: #rule-balance, #field-connected, #on_holding-thrust

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | ボタンを押すと蛇の頭が上方向に曲がり、離すと下方向に曲がる。各セグメントは前後のセグメントの動きに物理的に連動 |
| 終了条件 | 右から流れてくるアイテムを取り逃がして画面左端に到達させる |
| スコア | 通常アイテムを尻尾で取得すると蛇の長さ分、体で取得すると+1点。特殊アイテムを尻尾で取得すると長さの二乗分の点数とセグメント追加 |
| 難易度 | 時間経過でアイテムの移動速度が上昇し、セグメント間の連動も強化される |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| head(黒) | bar(nl,3) | ボタン操作で角度を変化、範囲制限あり | - |
| body(黒) | bar(nl,3) | 前のセグメントの動きに物理的に追従、バネのように連動 | 通常アイテム: +1点 / 特殊アイテム: -length点、セグメント削除 |
| tail(赤) | bar(nl,3) | 最後尾のセグメント、他と同様に連動 | 通常アイテム: +length点 / 特殊アイテム: +length²点、セグメント追加 |
| item(黒) | char(a) | 右から左へ移動、速度は徐々に加速 | 上記参照 |
| item(赤) | char(a) | 右から左へ移動、7個に1個出現 | 上記参照 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 蛇を上下に操作する直感的な操作。各セグメントの物理的な連動により、リアルな動きを表現。尻尾（赤）で取るべきという視覚的なヒント |
| (2) 視覚的フィードバック | セグメントの揺れ動く様子が視覚的に心地よい。赤いアイテムと赤い尻尾の対応関係が明確。取り逃しでXマークが表示 |
| (3) リスク・リワード | 特殊アイテムを尻尾で取れば大量得点＋強化だが、体で取るとペナルティ。長い蛇ほど操作が難しくなるが高得点の可能性も増す |
| (4) 斬新なメカニクス | セグメント間の物理的連動により、単なる軌跡追従ではない有機的な動き。どのセグメントで取るかによって結果が変わる戦略性 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #rule-balance | 通常アイテムは体で取り、特殊アイテムは尻尾で取るという判断が必要 | 尻尾を目標位置に持っていくための事前準備と、誤って体で取らないための慎重な操作が求められる |
| #field-connected | 各セグメントが物理的に連結し、前後のセグメントの動きに影響を受ける | 先頭の操作が即座に尻尾に伝わらず、時間差と慣性が戦略に影響。長くなるほど制御が難しくなる |
| #on_holding-thrust | ボタン長押しで蛇の頭を上方向に押し上げる | 押し続けると上に曲がり続け、離すと重力のように下に曲がる。リズミカルな押し引きが攻略の鍵 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 蛇を列車に変え、各車両がレールを曲がる動きとして表現。アイテムを乗客に変える |
| Combine (結合) | 複数の蛇を同時操作し、それぞれ異なる色のアイテムを担当させる協調ゲーム |
| Modify (修正) | セグメントごとに異なる色を付け、対応する色のアイテムのみ取得可能にする色合わせ要素を追加 |
