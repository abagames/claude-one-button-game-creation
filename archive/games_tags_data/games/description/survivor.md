# SURVIVOR (survivor)

**タグ**: #player-multiple, #on_pressed-jump

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | 9体のキャラクターを同時操作。タップで全員が同時ジャンプし、右から転がってくる樽を避ける。キャラ同士が接触すると頭の上に乗り、積み重なる |
| 終了条件 | 全てのキャラクターが樽に倒される |
| スコア | 樽が画面を通過するたびに、生き残っているキャラクター数分の得点 |
| 難易度 | 時間経過で樽の移動速度とキャラクターの動きが速くなる |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char(a/b) | タップで全員同時ジャンプ、中央に集まる力と樽から逃げる力が働く | - |
| player(乗る) | char(a/b) | 他のプレイヤーの頭上に固定、下のプレイヤーとともにジャンプ可能 | - |
| barrel | arc(半径可変) 黒 | 右から左へ転がる、サイズと速度はランダム | 触れたプレイヤーを倒し、画面外に吹き飛ばす |
| ground | rect(99,7) | 画面下部の床 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | タップで全員ジャンプという極めてシンプルな操作。複数キャラが同時に動くカオスと、それを一括制御する爽快感。積み重なる視覚表現が直感的 |
| (2) 視覚的フィードバック | ジャンプ時のパーティクル、樽の回転と地面の粉塵、倒されたキャラが吹き飛ぶアニメーション。キャラ同士が接触して積み重なる瞬間の効果音 |
| (3) リスク・リワード | 積み重なったキャラは高くジャンプできるが、まとめて倒されるリスクも。分散させて安全策を取るか、集めて高ジャンプを狙うかの判断 |
| (4) 斬新なメカニクス | 複数キャラの同時制御と、物理的な積み重ねによる動作変化。各キャラが独立した物理演算で動きながらも一括操作される独特のゲーム性 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-multiple | 9体のキャラクターを同時に操作 | 全員を生存させるのは困難。何体犠牲にしても生き残りを確保する判断力と、キャラ間の位置関係を把握する空間認識能力が必要 |
| #on_pressed-jump | タップで全キャラが同時にジャンプ | 全員が同じ入力を受けるため、個別の状態（床上/空中/他キャラの上）によって異なる結果が生まれる。タイミングの見極めが攻略の鍵 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 樽を複数の小さな敵キャラに変え、それぞれ異なる動きパターンで襲ってくるようにする |
| Combine (結合) | キャラ同士を紐で繋ぎ、1体が動くと他も引っ張られる物理パズル要素を追加 |
| Modify (修正) | 長押しの長さでジャンプ高度を変えられるようにし、より繊細な制御を可能にする |
