# BOTTOP (bottop)

**タグ**: player-inverted, field-auto_scroll, on_pressed-jump

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップでジャンプ / ホールドで飛行減速 |
| 挙動 | 画面上下に鏡像のキャラクターが同時存在。ジャンプで上下が逆に動き、赤いスパイクを回避 |
| 終了条件 | 上下いずれかのキャラクターがスパイクに接触 |
| スコア | 時間経過で自動的にスコア増加(difficulty/10) |
| 難易度 | 時間経過でスクロール速度(difficulty)増加、スパイクの出現頻度上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player_bottom | char("abc") | 下部、ジャンプで上昇 | スパイク接触で終了 |
| player_top | char("abc"反転) | 上部、ジャンプで下降 | スパイク接触で終了 |
| spikes | char("d"回転) | 上部/下部/中央に出現、左に流れる | 接触で終了 |
| ceiling/floor | rect(薄青) | 上下の境界 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 上下対称の視覚デザイン。黒いキャラと赤いスパイクの明確な色分け |
| (2) 視覚的フィードバック | ジャンプ時のキャラクター変化。上下対称の動きで状態が一目瞭然 |
| (3) リスク・リワード | ジャンプが両キャラに影響するため、片方を守るともう片方が危険に。判断の難しさ |
| (4) 斬新なメカニクス | 鏡像の同時操作。1つの入力で2つのキャラを逆方向に制御する独特のジレンマ |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| player-inverted | 上下に鏡像のキャラが同時存在 | 一つの入力で両方を制御。両方の安全を同時に確保する必要 |
| field-auto_scroll | 画面が自動スクロール | 常に前進し、休む暇がない。反射神経と判断速度が重要 |
| on_pressed-jump | タップでジャンプ | 上下逆の動きを計算し、両方が安全なタイミングを見極める |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 左右対称に変え、鏡像のような横方向の同時制御 |
| Combine (結合) | 3つ以上のキャラクターを異なる位置に配置し、複雑な同時制御 |
| Modify (修正) | 上下の重力を独立に制御できるモードを追加 |
