# LASER FORTRESS (laser_fortress)

**タグ**: weapon-wipe, on_holding-shoot, obstacle-penalty, rule-classify, field-1D, field-outpost

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | タップでレーザー照射開始、長押しで横方向に掃射 |
| 終了条件 | 味方にレーザー命中、または敵が左端到達 |
| スコア | 敵撃破で得点、連続撃破で倍率上昇（味方誤射で倍率リセット） |
| 難易度 | 時間経過で敵の移動速度と出現頻度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| fortress | 水色の基地 | 左端に固定 | - |
| sight | char("e") | 最も近い敵を自動追尾 | - |
| laser | 青いライン+紫ボックス | タップで照射開始、長押しで右方向に掃射 | - |
| enemy | char("c"/"d") 赤 | 右から左へ移動、一部は倍速 | レーザーで撃破+得点 |
| ally | char("a"/"b") 青 | 右から左へ移動 | レーザーで終了 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 赤=敵、青=味方の色分け、1次元移動で状況把握が容易 |
| (2) 視覚的フィードバック | レーザーのパーティクル、照準の自動追尾で敵位置を明示 |
| (3) リスク・リワード | 早期撃破で高得点だが味方誤射のリスク、タイミング判断が重要 |
| (4) 斬新なメカニクス | 自動追尾照準とレーザー掃射、敵味方の識別による戦略性 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| weapon-wipe | レーザーが横方向に掃射 | レーザーの照射開始位置と掃射範囲を計算、複数敵を連続撃破 |
| on_holding-shoot | 長押しでレーザー照射継続 | 長押し時間でレーザー到達範囲を調整、敵の位置予測が必要 |
| obstacle-penalty | 味方にレーザー命中でペナルティ | 敵と味方の識別、誤射を避けるタイミング制御 |
| rule-classify | 敵と味方を色で分類 | 瞬時の判断で敵味方を識別、混在時の優先順位決定 |
| field-1D | 横一列の移動 | 移動ラインが単純で予測しやすいが、密集時の判断が困難 |
| field-outpost | 基地防衛 | 敵の左端到達を阻止、基地を守る緊張感 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | レーザーを弾丸に変え、タップで単発射撃に |
| Combine (結合) | 複数レーンを追加し、上下にも照準を動かす2D防衛に |
| Modify (修正) | 味方を救出対象に変更し、レーザーで壁を破壊して救出 |
