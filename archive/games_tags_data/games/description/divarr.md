# DIVARR (divarr)

**タグ**: on_pressed-split, obstacle-penalty, on_pressed-turn, rule-control_weapons

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップで発射/分裂 |
| 挙動 | 1回目のタップで上向き弾を発射。以降のタップで各弾が左右に分裂 |
| 終了条件 | 人間(黄色キャラ)を撃つ、または障害物(赤キャラ)が地面に到達 |
| スコア | 障害物撃破時に弾数分。人間救出時に100点 |
| 難易度 | 落下物の速度・出現頻度増加、分裂角度調整 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player base | char(b) | 画面下部に固定 | - |
| shots | char(a) | 上下左右4方向のいずれかへ直進。タップで分裂 | - |
| obstacle | char(c) | 放物線軌道で落下 | +弾数点 |
| human | char(d) | 放物線軌道で落下。最初は画面左から警告付き | 撃つとゲームオーバー、救出で+100点 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 矢印弾、赤い障害物、黄色い人間という明確な色分け。「Don't hit me!」の警告テキストが親切 |
| (2) 視覚的フィードバック | 人間は最初に左側から出現し警告表示。画面上部では影表示で識別支援。ミスは赤い×マークで明確 |
| (3) リスク・リワード | 弾を多く分裂させると広範囲カバーだが制御困難に。分裂タイミングの判断が重要 |
| (4) 斬新なメカニクス | 「撃つべきでない対象」の存在。分裂弾の軌道が重なると相殺され消滅する独特の制約 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| on_pressed-split | タップで弾が分裂 | 弾数と制御性のトレードオフ。最大16発の制限により無限分裂を防止 |
| obstacle-penalty | 人間を撃つとペナルティ | ポジティブな目標(障害物破壊)とネガティブな制約(人間保護)の並存。注意力と識別力を要求 |
| on_pressed-turn | 分裂時に左右へ方向転換 | 弾道の予測と計画。4方向グリッドによる整然とした軌道パターン |
| rule-control_weapons | 発射後も弾を操作可能 | プレイヤーは「いつ分裂させるか」という時間的制御を持つ。発射して終わりではない継続的な判断 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 分裂を「弾の色変化」に置き換え、タップで色が変わり、同色の敵のみ撃破可能。人間は全色に反応 |
| Combine (結合) | rule:time_limitを追加し、人間は一定時間内に救出が必要。時間切れで失敗 |
| Modify (修正) | 16発超過時は最も古い弾から消滅。より多くの弾を維持する戦略とリスクを追加 |
