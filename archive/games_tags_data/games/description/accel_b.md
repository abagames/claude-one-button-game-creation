# ACCEL B (accel_b)

**タグ**: weapon-chase, on_pressed-shoot, rule-combo_multiplier, on_holding-move, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す / 長押し |
| 挙動 | 長押しで加速、タップで全敵/ミサイルに追跡ミサイルを一斉発射 |
| 終了条件 | 敵ミサイルに衝突 |
| スコア | 敵やミサイルを破壊するとmultiplier分のスコア獲得。破壊成功ごとに2倍(1→2→4→8→16→32→64)、新規ミサイル発射時に1にリセット |
| 難易度 | 時間経過で敵の移動速度・ミサイル速度・敵の発射頻度が上昇。プレイヤー位置に応じて画面が自動スクロール |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char('a', 青) | 長押しで加速(vx増加)、タップで追跡ミサイル一斉発射 | - |
| enemies | char('b', 紫) | 右から出現し左へ移動、定期的にミサイルを発射 | マルチプライヤー倍加+スコア |
| playerMissiles | bar(3x2, シアン) | 目標を追跡、到達/地面到達で爆発範囲ダメージ | - |
| missiles | bar(3x3, 黒) | プレイヤーを追跡、角度制限あり((3π/4)〜(5π/4)) | マルチプライヤー倍加+スコア(追跡ミサイル衝突時) / 終了(プレイヤー衝突時) |
| forests | box(緑) | 背景装飾、スクロール速度で流れる | - |
| smokes | box(淡色) | ミサイルの軌跡エフェクト、敵ミサイルは淡赤 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 基本図形(長方形・棒)と色分け(青=プレイヤー、紫=敵、シアン=味方、黒=敵弾)で視認性確保。操作は「[Tap] Fire / [Hold] Accel」のみ |
| (2) 視覚的フィードバック | 加速を紫バー(長さ=速度)で可視化、追尾軌跡を煙で表現、爆発時に粒子エフェクト。終了条件は敵ミサイル衝突のみで明確 |
| (3) リスク・リワード | 連続撃破でコンボ2倍ずつ増加(最大64倍)、ミサイル発射でリセット。「いつ攻撃するか」の判断が高スコアの鍵 |
| (4) 斬新なメカニクス | 単一ボタンで「加速移動」と「全敵/ミサイルへの一斉追尾攻撃」を同時制御。追尾アルゴリズムと加速感の融合 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| weapon-chase | 自動追尾ミサイルが敵を狙う | 発射タイミングとプレイヤー位置で連続撃破を狙う戦略性 |
| on_pressed-shoot | タップで全敵/ミサイルに一斉発射 | タイミングよくタップしてコンボ維持、攻撃判断が高得点に直結 |
| rule-combo_multiplier | 連続撃破で2倍ずつ増加(最大64倍)、発射でリセット | コンボ維持の集中力とリスク管理がゲームの核心 |
| on_holding-move | 長押しで加速、スクロールに合わせて右へ進む | 加速量を微調整し、回避や射撃位置確保。操作の継続性が生存を左右 |
| field-auto_scroll | プレイヤー位置に応じて自動スクロール | スクロール速度に合わせた加速タイミングが生存とスコアに直結 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 追尾ミサイルを「レーザー」や「バリア」に変更。追尾ではなく範囲攻撃で戦略一新 |
| Combine (結合) | 加速メカニクスに「障害物回避」を追加。加速しながら障害物を避けつつ敵を倒す複雑な操作バランス |
| Modify (修正) | プレイヤーを巨大な「棒状」に変更し、加速時のヒットボックスを拡大。回避戦略が根本的に変化 |
