# M RIDER (m_rider)

**タグ**: #rule-chain, #on_holding-thrust, #field-bottomless, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | 飛来するミサイルに飛び乗り、上下操作で敵機を撃墜、連鎖破壊を狙う |
| 終了条件 | ミサイルに乗った状態で画面上端を超える、または画面下端に落ちるとゲームオーバー |
| スコア | 敵機撃墜で+multiplier点、連鎖するごとにmultiplier増加、ミサイル乗り換え時にmultiplier半減 |
| 難易度 | ミサイルと敵機の速度が時間経過で増加、敵機の出現頻度も上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| rider | char(a/b) | ミサイル搭乗中は上下制御、落下中は重力で落下＋回転 | 画面上端/下端→終了 |
| missile | box(18,3) | 右から左に放物線軌道、Hold時に上昇 | 搭乗可能、敵機接触→ライダー射出＋敵機撃墜 |
| plane | char(c) | 右から左に直線移動、撃墜されると9tickの爆発タイマー | ミサイル接触→撃墜、連鎖で周囲36pxの敵機も撃墜 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | ミサイル（赤）と敵機（黒）の明確な区別。パーティクルでミサイルの推進を視覚化 |
| (2) 視覚的フィードバック | ミサイル乗り換え時に「powerUp」音、敵機撃墜で「jump」音、連鎖完了で「coin」音。パーティクルと得点表示で連鎖の達成感 |
| (3) リスク・リワード | multiplierを維持するには1つのミサイルで連続撃墜を狙うが、ミサイルは有限で画面外に消える。乗り換えるとmultiplier半減だが新たな機会を得る |
| (4) 斬新なメカニクス | プレイヤーが「攻撃手段そのもの」に乗る構造。ミサイルは消耗品で、乗り換えのタイミングが戦略の核心。連鎖範囲を計算し、密集した敵機群を一掃する爽快感 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #rule-chain | 敵機撃墜時に周囲36pxの敵機に連鎖、連鎖係数0.9で減衰 | 敵機の配置を先読みし、最も密集した地点で撃墜開始する判断力が高得点の鍵 |
| #on_holding-thrust | ミサイル搭乗中はHoldで上昇、離すと下降 | 上下の微調整が連鎖の成否を分ける。敵機の高度を把握し、最適な軌道を描く技術 |
| #field-bottomless | 画面下端に落ちると即死 | ミサイルを失うと重力で落下し、次のミサイルを捕まえられないと死亡。緊張感の源泉 |
| #field-auto_scroll | プレイヤーが右に進むと画面スクロール | 前進し続けることが要求され、後退して安全を確保する選択肢がない |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ミサイルを「サーフボード」に変更し、波に乗って障害物を避けるサーフィンゲームに |
| Combine (結合) | 複数のミサイルタイプ（高速・直進/低速・曲線）を追加し、状況に応じた乗り換えの戦略性を強化 |
| Modify (修正) | ミサイルの燃料ゲージを追加し、一定時間後に墜落する制約。より頻繁な乗り換えを強制 |
