# UP 1 WAY (up_1_way)

**タグ**: #field-auto_scroll, #field-lanes, #on_pressed-jump, #on_got_item-power_up

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップ：上の階層にジャンプ（上にしか移動できない） |
| 挙動 | 6階層の床を自動で走る。タップで上にジャンプし、穴があると自動的に下に落下。竹を集めてスカルを避ける |
| 終了条件 | スカルに接触 |
| スコア | 竹を取ると+1点 |
| 難易度 | 時間経過でスクロール速度が上昇し、スカルの出現頻度も増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player(a) | char("a") アニメ 黒色 | 各階層を自動的に右に走る。穴があると下の階層に自動落下 | - |
| floors | rect 緑/薄黒 | 6階層の床。各階層に不規則な穴が開いており、左にスクロール | - |
| bamboo(c) | char("c") 黄色 | 各階層にランダム出現。連続して出現することもある | +1点 |
| skull(d) | char("d") アニメ 赤色 | 各階層にランダム出現。プレイヤーに接触で終了 | 終了 |
| power-up(h) | char("h") 黄色 | 稀に出現。取得すると全てのスカルが竹に変換 | 全スカルを竹に変換 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 緑の床、黄色の竹、赤いスカルという明確な色分け。上にしか移動できないという制約が独特の戦略を生む |
| (2) 視覚的フィードバック | ジャンプで「jump」音、落下で「click」音。竹取得で「coin」音、パワーアップで「powerUp」音。スカル接触で「explosion」音 |
| (3) リスク・リワード | 上にジャンプして竹を取るか、下に留まって安全を確保するか。パワーアップで一時的に無敵化し高スコアチャンス |
| (4) 斬新なメカニクス | 上にしか移動できない制約。穴による強制的な下降と、タップによる上昇の組み合わせが独自のリズムを生む |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-auto_scroll | 床と障害物が自動的に左にスクロール | 一定のペースで判断を迫られ、タイミングを逃すと選択肢が狭まる。テンポ感が緊張を生む |
| #field-lanes | 6階層の床に分かれたフィールド | どの階層にいるかが生存と得点の鍵。階層間の移動タイミングを計算する戦略性 |
| #on_pressed-jump | タップで上の階層にジャンプ（上にしか移動できない） | 上への移動のみという制約が、穴の位置を利用した下降と組み合わさり独特の攻略を生む |
| #on_got_item-power_up | パワーアップ取得で全スカルが竹に変換 | 一時的な安全とスコア獲得チャンス。パワーアップを狙うか無視するかの戦略的選択 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 上移動を下移動に変更し、天井に穴がある設定に。または左右移動に変更して横スクロールアクション化 |
| Combine (結合) | 敵キャラクターを追加。階層間を移動する敵を避けながら竹を集める要素を加える |
| Modify (修正) | 階層数を増減。4階層で簡単に、8階層で難しく。または階層の高さをランダムに変動させる |
