# METEO PLANET (meteo_planet)

**タグ**: #field-space, #rule-proximity_bonus, #obstacle-fall, #field-rotate

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | 惑星の周りを90度ずつ回転移動し、4方向から落下する隕石を避けつつ宝石を回収 |
| 終了条件 | 隕石に接触するとゲームオーバー |
| スコア | 宝石回収で+サイズ点（1-6点） |
| 難易度 | 隕石と宝石の落下速度が時間経過で増加、出現頻度も上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char(a/b) | Tapで惑星の周りを90度回転、回転中はアニメーション | - |
| meteorite | char(c) | 4方向から惑星に向かって直線落下 | 終了 |
| gem | box(1-6) | 隕石の両側に配置され、惑星に向かって落下 | +サイズ点 |
| planet | arc | 画面中央の惑星、プレイヤーの回転軸 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 黒＝危険（隕石）、黄色＝報酬（宝石）という色分け。惑星を中心とした回転構造が空間認識を容易にする |
| (2) 視覚的フィードバック | Tap時に「select」音、宝石回収で「powerUp」音、隕石接触で「explosion」音。回転アニメーションでプレイヤーの移動を明示 |
| (3) リスク・リワード | 大きな宝石ほど高得点だが、隕石の周辺に配置されるため回収リスクが高い。回転のタイミングが早いと宝石を逃し、遅いと隕石に接触 |
| (4) 斬新なメカニクス | 円形の移動空間で、4方向すべてが等しく重要。従来の左右移動ではなく、90度回転という離散的な移動が独特のリズムを生む |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #field-space | 宇宙空間を背景に、惑星の周りを回転する設定 | 360度すべての方向に注意を向ける必要があり、従来の左右型ゲームとは異なる空間認識 |
| #rule-proximity_bonus | 宝石のサイズ（1-6）に応じて得点が変化 | 中央の大きな宝石を狙うか、外側の小さな宝石で安全に得点するかの判断 |
| #obstacle-fall | 隕石が惑星に向かって落下 | 4方向すべてから落下するため、次にどの方向に移動すべきか常に判断が必要 |
| #field-rotate | プレイヤーが惑星の周りを90度ずつ回転 | 回転には時間がかかるため、次の危険を予測して早めに移動開始する先読みが重要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 惑星を「塔」に変更し、4方向ではなく8方向（45度刻み）の回転で、より細かい位置調整を要求 |
| Combine (結合) | 隕石が惑星に到達すると爆発し、近くのプレイヤーにもダメージを与える範囲攻撃を追加 |
| Modify (修正) | 連続Tapで回転速度を上げられるが、回転中に隕石に接触するリスクも増加する仕組み |
