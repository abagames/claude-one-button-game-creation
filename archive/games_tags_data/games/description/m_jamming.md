# M JAMMING (m_jamming)

**タグ**: #weapon-explosion, #rule-chain, #player-automatic, #obstacle-chase, #on_holding-extend, #on_released-erase, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し / 離す |
| 挙動 | 追跡ミサイルの群れから自動回避しながら、範囲を拡大しジャミングで反射、連鎖爆発を誘発 |
| 終了条件 | ミサイルに接触するとゲームオーバー |
| スコア | ジャミング時に範囲内のミサイル数だけ得点、連鎖爆発でmultiplier加算 |
| 難易度 | ミサイルの出現数・追跡速度・範囲拡大速度が時間経過で増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| robot | char(a) | 自動的に最も近いミサイルから逃げる、Hold時は移動速度低下 | ミサイル接触→終了 |
| missile | bar(3,3) | プレイヤーを追跡、ジャミング後は反射して直進（cyan化） | プレイヤー接触→終了、爆発/cyanミサイル接触→爆発＋得点 |
| explosion | arc | 拡大後縮小する爆発範囲、cyanミサイルが接触すると連鎖 | - |
| jamming_range | arc | Hold時に拡大、Release時にジャミング発動 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 紫＝敵ミサイル、cyan＝ジャミング済み、赤＝爆発という色分け。円の拡大でジャミング範囲が視覚化 |
| (2) 視覚的フィードバック | Hold時に「select」音、Release時に「laser」音、連鎖爆発で「powerUp」音。パーティクルとスコア表示で連鎖の達成感 |
| (3) リスク・リワード | 範囲を大きくするほど多数のミサイルをジャミングできるが、Hold時は移動速度が低下し危険。連鎖を狙うとmultiplier増加だが、タイミングを誤ると被弾 |
| (4) 斬新なメカニクス | プレイヤーが自動回避し、プレイヤーの役割は「いつジャミングするか」のタイミング判断のみ。敵を無力化するのではなく、反射して連鎖の弾に変える戦略性 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #weapon-explosion | ジャミング済みミサイルが爆発や他のcyanミサイルに接触すると連鎖爆発 | 連鎖の配置を考え、1回のジャミングで複数の爆発を誘発する戦略 |
| #rule-chain | cyanミサイル→爆発→他のcyanミサイルという連鎖反応 | multiplierを最大化するため、密集したミサイル群を一斉にジャミングする判断 |
| #player-automatic | プレイヤーが自動的に最も近いミサイルから逃げる | 移動を考えず、ジャミングのタイミングのみに集中できる |
| #obstacle-chase | ミサイルが追跡してくるため、逃げ続けても安全地帯はない | 常に危険が迫り、ジャミングのタイミングが生存の鍵 |
| #on_holding-extend | Hold時間に応じて範囲が拡大 | どこまで範囲を広げるかの判断。欲張ると移動速度低下で被弾リスク |
| #on_released-erase | Release時にジャミング発動、範囲内のミサイルを反射＋multiplierリセット | タイミングが全て。早すぎると少数しかジャミングできず、遅すぎると被弾 |
| #field-auto_scroll | 背景の星がプレイヤーの移動に応じてスクロール | 空間の広がりを演出、プレイヤーの移動感を視覚化 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | ミサイルを「敵の弾」から「味方ユニット」に変更し、ジャミングで操作権を奪う戦略ゲームに |
| Combine (結合) | 複数の範囲タイプ（小範囲・高速/大範囲・低速）を切り替え可能にし、状況判断の要素を追加 |
| Modify (修正) | ジャミング後のミサイルの方向をプレイヤーが制御でき、より直接的な連鎖の構築が可能に |
