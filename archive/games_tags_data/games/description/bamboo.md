# BAMBOO (bamboo)

**タグ**: on_holding-avoid, player-reflect, field-1D, obstacle-well_up

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | タップで方向転換 / ホールドで竹を通過 |
| 挙動 | 左右に移動し、地面から伸びる竹を適切な高さで刈り取る。伸びすぎた竹に当たると反射して加速 |
| 終了条件 | 竹が画面上端(高さ89)に到達 |
| スコア | 竹を高さ5-25で刈り取ると得点。高さに比例して得点増加(高さ/3)^2。最大10点 |
| 難易度 | 時間経過で竹の成長速度と出現頻度増加。一部は2倍速で成長 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("ab") | 左右に移動、減衰する速度 | - |
| bamboo_sprout | box(芽、薄黄) | 高さ0-5、ゆっくり成長 | 通過可能 |
| bamboo_young | box(黄~縞) | 高さ5-25、刈り取り可能 | 刈り取って得点 |
| bamboo_mature | box(緑) | 高さ25+、硬い | 反射+加速 |
| floor | rect(紫) | 下部に固定 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 竹の成長段階を色で表現(黄→縞→緑)。高さと色で刈り取りタイミングが明確 |
| (2) 視覚的フィードバック | 刈り取り時のスコア表示。反射時のパーティクルと加速感。成長の視覚的変化 |
| (3) リスク・リワード | ギリギリまで伸ばすほど高得点だが、反射のリスク。反射は加速で新たなチャンス |
| (4) 斬新なメカニクス | 敵を回避するのではなく、適切なタイミングで収穫するという発想の転換 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| on_holding-avoid | ホールドで竹を通過 | 刈り取りと通過の選択。ホールドのタイミングが重要 |
| player-reflect | 硬い竹に当たると反射 | ミスがペナルティだけでなく、加速という新たな状況を生む |
| field-1D | 横一列の移動のみ | シンプルな移動で竹の管理に集中できる |
| obstacle-well_up | 障害物が下から湧き出る | 障害物が成長する時間軸が戦略に影響 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 竹を作物に変え、水やりや肥料の要素を追加 |
| Combine (結合) | 複数種類の植物を追加し、収穫順序や組み合わせで得点変化 |
| Modify (修正) | 竹の色で成長速度や得点が異なる品種システム |
