# FLOORS 5 (floors_5)

**タグ**: rule-match, on_pressed-jump, player-paint, field-bottomless

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（ジャンプ） / 長押し（飛行中の上昇速度調整） |
| 挙動 | 5色の床の間をジャンプし、着地した床を塗っていく。5色すべてに着地すると倍率が上昇 |
| 終了条件 | 画面下（y > 99）に落下 |
| スコア | 床から離れるときに「塗った幅 × 倍率」。床全体を塗ると3倍ボーナス |
| 難易度 | 時間経過でスクロール速度が増加（difficulty * 0.02） |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| car（プレイヤー） | char("a") 6x6 | 重力で落下、タップでジャンプ、長押しで上昇速度減少、2段ジャンプ可能 | - |
| floors（床） | rect(width, 6) | 5色（red/green/blue/yellow/purple）の床が左にスクロール、画面外に出るとランダム位置に再配置 | 着地時: 速度リセット、床を塗る、ジャンプカウントリセット |
| paint（塗り） | rect(paintTo - paintFrom, 4) | 床上の塗られた部分を色で表示 | - |
| color markers | char("c"/"d") | 画面下部に5色のマーカー表示（着地済み/未着地） | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 5色の床を視覚的に区別し、画面下部のマーカーで進捗を明示。「すべての色に着地する」というルールがテキストなしで理解できる |
| (2) 視覚的フィードバック | 床を塗る視覚表現、画面下部の着地マーカー、倍率表示、スコア表示位置（床から離れる時）が明確。ゲームオーバーは画面下への落下で明白 |
| (3) リスク・リワード | 床全体を塗るとボーナス3倍、5色コンプリートで倍率上昇。遠い床や小さい床への挑戦がリスク・リワードを生む |
| (4) 斬新なメカニクス | 「色マッチング」と「ペイント」を組み合わせた独自システム。床を塗る範囲が得点に直結し、完全塗りつぶしでボーナス。2段ジャンプと長押し飛行の組み合わせによる精密な軌道制御 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| rule-match | 5色すべての床に着地すると倍率が1増加 | プレイヤーは安全な床だけでなく、リスクのある遠い床や小さい床にも挑戦する動機付け。長期的なスコア戦略が重要 |
| on_pressed-jump | タップでジャンプ、空中で2段ジャンプ可能 | 床から床への移動手段。2段ジャンプにより精密な軌道調整が可能。タイミングとジャンプ回数の管理が習熟要素 |
| player-paint | 床上を移動すると塗った範囲が記録され、離れるときにスコア化 | 床を長く塗るほど高得点。床全体を塗ると3倍ボーナス。どこでジャンプするか、どれだけ塗るかの判断が戦略的 |
| field-bottomless | 画面下に落ちるとゲームオーバー | 高さ管理が必須。ジャンプと飛行のタイミングミスは即死に繋がり、緊張感を生む |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 床を塗る代わりに「敵を倒す」「アイテムを集める」に置き換え、5種類のターゲットをコンプリートする仕組みに |
| Combine (結合) | 床の色をプレイヤーが選択できる「ペイントガン」を追加。塗る色を戦略的に選べるようにする |
| Modify (修正) | 床が回転したり、上下に動いたりする動的な床に変更。塗る難易度が大幅に上昇 |
