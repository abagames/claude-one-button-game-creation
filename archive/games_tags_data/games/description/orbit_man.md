# ORBIT MAN (orbit_man)

**タグ**: #player-rotate, #on_pressed-go_forward, #field-space, #field-pins, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す（次の惑星へ発射） |
| 挙動 | 惑星の周りを回転し、回転棒(bar)を伸ばす。ボタンを押すと棒が貫通している惑星のうち最も遠い惑星へ移動し、貫通した惑星を破壊 |
| 終了条件 | 発射時に貫通している惑星がない、または現在の惑星が画面下端を超える |
| スコア | 惑星を破壊するとmultiplier点獲得、破壊するごとにmultiplier+1、発射時はmultiplier-1 |
| 難易度 | difficultyに応じて回転速度とスクロール速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| man | 人型(cyan, line) | 惑星の周りを回転、発射で次の惑星へ移動 | - |
| bar | line(light_blue) | manから伸びる長い棒、惑星を貫通して到達可能性を示す | - |
| planets | arc(black) | 上から出現し下へスクロール、半径4-9のランダムサイズ | manの惑星と衝突で終了、bar貫通で破壊対象に |
| stars | rect(light_black, 1px) | 背景の星、スクロールで宇宙空間の移動を表現 | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 宇宙空間(dark theme)に浮かぶ黒い惑星と回転する青い人という直感的なビジュアル、棒で到達可能な惑星が明確 |
| (2) 視覚的フィードバック | 惑星破壊時のパーティクルエフェクト、効果音で成功・失敗を明示、multiplier表示で連鎖ボーナスを視覚化 |
| (3) リスク・リワード | 複数の惑星を一度に貫通すれば高得点だが、最遠の惑星への移動距離が長くなり次の発射までの時間が短縮される |
| (4) 斬新なメカニクス | 回転と貫通判定の組み合わせ。惑星間を"渡り歩く"という宇宙空間ならではの移動感覚が独特 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-rotate | 惑星の周りを自動で回転（角度+=difficulty*0.03） | 回転のタイミングを見計らって、複数の惑星を貫通する角度で発射する必要がある |
| #on_pressed-go_forward | ボタンを押すと回転方向に発射され、最も遠い貫通惑星へ移動 | 発射タイミングが重要、早すぎても遅すぎても目標の惑星を貫通できない |
| #field-space | 宇宙空間を模した暗いテーマ、星が背景に流れる | 宇宙空間の無重力感を演出、惑星間を渡り歩く独特の移動感 |
| #field-pins | 惑星が画面上に複数配置され、それぞれが"足場"として機能 | 惑星の配置を予測し、次の移動先を計画する戦略性が求められる |
| #field-auto_scroll | 下方向に自動スクロール、プレイヤーの高度で速度調整 | 一定の緊張感を維持しつつ、惑星間の移動に集中できる |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 惑星を「浮島」に置き換え、ファンタジー世界の空中移動ゲームに変化させる |
| Combine (結合) | 重力の概念を追加し、惑星の質量に応じた引力で軌道が変化する要素を加える |
| Modify (修正) | 回転速度をボタン長押しで制御可能にし、より精密なタイミング調整を可能にする |
