# TWHOLS (twhols)

**タグ**: #player-reflect, #on_pressed-turn, #field-holes

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す：左右反転 |
| 挙動 | 斜め45度で移動する緑のボックス。壁の穴を通り抜け、壁に当たると上下反転しスコア獲得 |
| 終了条件 | 画面の上端または下端に到達 |
| スコア | 壁に当たって上下反転する度に+1点 |
| 難易度 | 時間経過で移動速度が上昇し、壁の移動速度も増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | box(5) 緑色 | 斜め45度で等速移動。画面左右端で自動的に左右反転 | - |
| walls | rect(-99,5) + rect(99,5) 青色 | 中央に穴が開いた上下2本の壁。左に自動スクロール。画面左端を過ぎると右端に再出現し速度増加 | 上下反転 + スコア+1 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 緑のボックスと青い壁という最小限の要素。斜め移動と穴を通り抜けるというルールが視覚的に明確 |
| (2) 視覚的フィードバック | 壁との衝突で「hit」音とスコア加算。画面端到達で「lucky」音と共にゲームオーバー。成功と失敗が音と視覚で即座に分かる |
| (3) リスク・リワード | 壁に当たることがスコア源だが、穴を通り抜け損ねると軌道が変わり危険な位置に。タイミングを狙ってわざと当たる選択も可能 |
| (4) 斬新なメカニクス | 壁との衝突が得点源という逆転の発想。穴の位置と移動軌道の予測が攻略の鍵となる独自性 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-reflect | 壁および画面左右端で反射する斜め移動 | 反射の軌道計算が攻略の核心。穴の位置と自分の軌道を予測し、安全な反射か危険な反射かを判断 |
| #on_pressed-turn | タップで左右の移動方向を反転 | 穴を通り抜けるタイミング調整や、壁に当ててスコアを稼ぐかの選択。一度の入力が長期的な軌道に影響 |
| #field-holes | 中央に穴が開いた壁が左にスクロール | 穴の通過が生存の鍵。壁に当たるとスコアだが軌道が変わるため、穴を狙うか壁を狙うかの戦略的選択 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 壁を移動する敵キャラクターに置き換え、穴の代わりに安全地帯を設ける。または壁を上下にも移動させて予測をさらに困難に |
| Combine (結合) | アイテム収集要素を追加。穴の中にコインを配置し、穴を通り抜けることで追加スコア。壁を避ける行動に報酬を与える |
| Modify (修正) | 穴のサイズをランダムまたはスコアに応じて変動。ハイスコア時に穴が狭くなり難易度上昇。または壁の数を3本、4本と増やす |
