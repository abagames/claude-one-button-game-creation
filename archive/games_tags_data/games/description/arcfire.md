# ARCFIRE (arcfire)

**タグ**: weapon-arc, field-outpost, on_holding-adjust_angle, player-rotate, on_released-throw

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | ホールドで弧の範囲を設定 / リリースで発射 |
| 挙動 | 中央の円形エリアを守る砲台。回転する照準で扇形範囲を決め、弧状弾幕を発射。反動で自機も移動 |
| 終了条件 | 敵が自機または中央の青い円に接触 |
| スコア | 弧状弾幕で敵を撃破。連続ヒットでmultiplier増加 |
| 難易度 | 時間経過で照準回転速度上昇、敵の出現頻度・移動速度増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("ab") | 中央エリア内を移動、回転照準 | 敵接触で終了 |
| shots | arc(弧状) | 弧に沿って拡大していく弾幕 | - |
| enemies | char("de") | 外周から中央に向かって移動 | 弾幕で撃破、+スコア |
| center | arc(円) | 中央の防衛対象エリア | 敵接触で終了 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 回転する照準線と広がる弧で射撃範囲を視覚化。緑→青→赤の色分けで状態明確 |
| (2) 視覚的フィードバック | ホールド中に緑の弧が広がる予測線。発射で青い弾幕。敵撃破でパーティクル |
| (3) リスク・リワード | 広い弧=多くの敵を狙えるが反動も大きく制御困難。狭い弧=精密だが取りこぼしリスク |
| (4) 斬新なメカニクス | 弧状の範囲攻撃と反動移動の組み合わせ。攻撃が移動手段にもなる二重性 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| weapon-arc | 弧状の範囲攻撃 | 複数の敵を同時に狙える面制圧だが、角度調整が重要 |
| field-outpost | 中央エリアを防衛 | プレイヤーの移動範囲が制限され、守りの戦略が求められる |
| on_holding-adjust_angle | ホールドで射撃角度を調整 | タイミングと角度の両方を同時に判断する必要がある |
| player-rotate | プレイヤーの照準が回転 | 常に動く照準に合わせる反射神経と予測が必要 |
| on_released-throw | リリースで発射 | ホールド時間が攻撃範囲を決定。溜め動作的な緊張感 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 弧を直線レーザーに変え、貫通攻撃に |
| Combine (結合) | 複数の砲台を配置し、連携防衛 |
| Modify (修正) | 弧の大きさで威力と反動が変化するトレードオフを追加 |
