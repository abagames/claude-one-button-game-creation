# LIFT UP (lift_up)

**タグ**: #on_got_item-reverse_state, #on_holding-move, #field-spike, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | プレイヤーは自動的に左右に往復移動。長押しで画面の上昇速度が加速し、壁の隙間を素早く通り抜ける。ターンアイテムで方向反転、ボーナスアイテムでスコア倍率上昇。 |
| 終了条件 | 壁の赤いスパイクに接触、または画面上部に到達 |
| スコア | ボーナスアイテム取得でmultiplier × 得点。取り逃すとmultiplierが減少 |
| 難易度 | 時間経過で壁の幅・位置がランダムに変動し、通路が狭まる |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char(a,b) | 左右に自動往復移動、長押しで上昇加速 | - |
| walls | char(c)+rect | 上から下へスクロール、幅・位置が変動 | 終了 |
| turn item | char(d) cyan | スクロールに追従、壁に接触で反発 | 水平移動方向反転 |
| bonus item | char(e) yellow | スクロールに追従、壁に接触で反発 | +multiplier点、multiplier増加 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | キャラクターと壁、2種類のアイテムのみ。色分け（赤=危険、cyan=方向転換、yellow=ボーナス）で役割が即座に理解できる。 |
| (2) 視覚的フィードバック | スパイク接触で即座にゲームオーバー。アイテム取得で効果音と視覚的な得点表示。ボーナス取り逃しでmultiplier減少という明確なペナルティ。 |
| (3) リスク・リワード | ボーナスアイテムを取らないとmultiplierが減る仕組みで、危険な位置でも取りに行く必要がある。長押しで速く上昇するが制御が難しくなる。 |
| (4) 斬新なメカニクス | 「長押しで速く進む」という逆転の発想。通常は慎重に進むべき場面で、あえて加速することでリスクとスキルが問われる。 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #on_got_item-reverse_state | ターンアイテム取得で水平移動方向を反転 | 壁との位置関係を考慮し、適切なタイミングで方向転換する判断力が求められる |
| #on_holding-move | 長押しで画面の上昇速度が加速（最大4倍） | 危険な場所を素早く通過するか、慎重に進むかの判断。習熟がスコアに直結 |
| #field-spike | 壁の両端に赤いスパイク配置 | 通路の幅が常に変動し、プレイヤーの移動範囲を制限。接触即死で緊張感を生む |
| #field-auto_scroll | 画面が自動的に上方向にスクロール | プレイヤーは常に前進を強制され、立ち止まる選択肢がない。長押しで加速可能 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | スパイクを動く障害物や敵に置き換え、回避の難易度を上げる |
| Combine (結合) | ジャンプ要素を追加し、上下の移動も制御可能にする |
| Modify (修正) | 長押し時間に応じてプレイヤーのサイズが変化し、通過できる隙間が変わる |
