# LADDER DROP (ladder_drop)

**タグ**: player-automatic, player-scaffold, on_pressed-fall, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | 上部のパネルをタップで落下、プレイヤーは梯子で登り降りしながら自動左右歩行 |
| 終了条件 | プレイヤーが画面下に落下、または落下中のパネルに接触 |
| スコア | コイン取得で得点と倍率上昇、取り逃すと倍率減少 |
| 難易度 | 時間経過でパネルの移動速度と落下速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| player | char("a"/"b"/"c") | 自動で左右歩行、梯子で登り降り、壁や床で反転 | 落下パネルで終了 |
| panel | char("e"=梯子, "f"=床) | 上部で左右移動、タップで落下、着地後固定 | - |
| coin | char("i") | パネル上に配置（4パネルごと） | +得点、倍率+1 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 梯子と床の明確な区別、自動歩行で操作を簡略化 |
| (2) 視覚的フィードバック | パネルの落下と積み重なり、梯子の登り降りアニメーション |
| (3) リスク・リワード | コイン取得のための梯子配置読み、パネル落下タイミングの判断 |
| (4) 斬新なメカニクス | 自動歩行とパネル落下の組み合わせ、梯子による垂直移動の戦略性 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| player-automatic | プレイヤーが自動で左右歩行 | 操作を梯子の登り降りとパネル落下に集中、歩行パターンの予測が必要 |
| player-scaffold | 梯子で登り降り可能 | 梯子の配置を読み、垂直移動でコインや安全地帯へ移動 |
| on_pressed-fall | タップでパネル落下 | パネル落下のタイミングと位置を調整、梯子配置を戦略的に構築 |
| field-auto_scroll | 画面が下方向にスクロール | 最下部への落下を避け、上部への移動を継続 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 梯子をエレベーターやロープに変え、連続移動の仕組みに |
| Combine (結合) | パネルに色を追加し、同色パネル上でのみコイン取得可能に |
| Modify (修正) | タップ長押しでパネル落下速度を調整、精密な配置が可能に |
