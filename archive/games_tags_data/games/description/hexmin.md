# HEXMIN (hexmin)

**タグ**: #field-outpost, #player-multiple, #on_pressed-turn

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | タップで矢印を60度回転、中心に迫る敵を狙った方向で消去 |
| 終了条件 | 矢印が向いていない方向で敵が中心に到達 |
| スコア | 敵を消去すると得点、連続消去で倍率が上昇（最大+6ずつ） |
| 難易度 | 時間経過で敵の出現頻度と接近速度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| arrow | 六角形の2方向を赤く表示 | タップで60度回転 | - |
| enemy | 六角形（緑→青に変色） | 外側から中心へ段階的に接近（5段階） | 矢印方向一致で+得点、不一致で終了 |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 六角形の幾何学的配置と色分け（緑→青で危険度上昇）で状況を一目で把握可能 |
| (2) 視覚的フィードバック | 敵の段階的接近と色変化で危険を明示、矢印の赤色で狙い位置を強調 |
| (3) リスク・リワード | 連続消去で倍率上昇だがタップでも倍率減衰、タイミング判断がスコアに直結 |
| (4) 斬新なメカニクス | 六角形配置の6方向レーンシステム、回転する矢印で複数脅威を管理 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| field-outpost | 中心から6方向への放射状レーン | 全方位への注意分散と優先順位判断が必要 |
| player-multiple | 2方向同時に狙える矢印 | 効率的な回転で複数の脅威を連続処理できる |
| on_pressed-turn | タップで60度回転 | 最小回転で狙いを定める効率性と、回転による倍率減衰のトレードオフ |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 六角形を四角形や八角形に変え、方向数を4/8に変更 |
| Combine (結合) | 各レーンに異なる色の敵を配置し、色マッチングルールを追加 |
| Modify (修正) | 矢印が自動回転し、タップで一時停止させる逆転操作 |
