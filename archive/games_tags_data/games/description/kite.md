# KITE (kite)

**タグ**: field-weather, player-string, field-spike, field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 長押し |
| 挙動 | 長押しで風を吹かせ凧を右上に押し上げ、紐で繋がったプレイヤーを引っ張る |
| 終了条件 | プレイヤーが上下の棘に接触 |
| スコア | 棘を避けると継続得点、コイン取得でボーナス（プレイヤー速度×5×50点） |
| 難易度 | 時間経過で棘の高さと頻度が増加 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| kite | box(6,6) | 長押しで右上に移動、重力で落下、紐の張力で制約 | 棘で跳ね返り |
| player | char("a"/"b") | 凧に紐で繋がり引っ張られる、地面を歩く | 棘で終了 |
| spike | line（赤） | 上下から周期的に生える、2つの正弦波で高さ変化 | - |
| coin | char("d") | 右から左へ移動、プレイヤーに近づくと引き寄せられる | +速度依存ボーナス |
| wind | char("c") | 左から右へ移動、長押しで頻度増加 | 凧を押す |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | 凧と紐の物理的繋がり、風の視覚化で操作を直感的に理解 |
| (2) 視覚的フィードバック | 凧と紐の動的な張力表現、棘の周期的な生成パターン |
| (3) リスク・リワード | コイン取得で高得点だが速度上昇でリスク増、風の強さ調整で制御 |
| (4) 斬新なメカニクス | 紐で繋がった2オブジェクトの間接制御、風による凧の操作 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| field-weather | 風が凧を押す環境要素 | 長押しで風の強さを調整、凧の高さと速度をコントロール |
| player-string | 凧とプレイヤーが紐で繋がる | 紐の張力バランス、凧の位置がプレイヤーの動きに影響 |
| field-spike | 上下から棘が生える | 正弦波パターンの読み取り、凧の高さ調整が必要 |
| field-auto_scroll | 凧の位置に応じてスクロール | 前進を維持しつつ棘を避ける位置取り |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 風を重力に変え、タップで重力方向を反転 |
| Combine (結合) | 複数の凧を紐で繋ぎ、連鎖的な動きを制御 |
| Modify (修正) | 紐の長さを動的に変更できるようにし、距離調整の戦略性を追加 |
