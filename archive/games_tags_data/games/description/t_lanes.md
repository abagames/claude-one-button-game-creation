# T LANES (t_lanes)

**タグ**: #player-multiple, #field-lanes, #on_got_item-spawn, #on_pressed-reverse_state, #field-auto_scroll

## 1. コア・メカニクス

| 項目 | 内容 |
|:---|:---|
| 入力 | 押す |
| 挙動 | 複数の車を同時操作し、5本のレーンを走行。タップで全交差点の方向指示を切り替え（直進⇔レーン変更）、車は交差点を通過すると指示に従って進路変更 |
| 終了条件 | 全ての車が停止標識に衝突または画面外に消える |
| スコア | 交差点を通過するたびに現在の車数分の得点。新しい車が既存の車に触れると×10点 |
| 難易度 | 時間経過で車の速度が上昇 |

## 2. オブジェクト仕様

| 名前 | 形状 | 動作 | 衝突時 |
|:---|:---|:---|:---|
| car(動) | char(a/b) 黒 | 交差点の指示に従って直進またはレーン変更、自動スクロールに追従 | - |
| car(停) | char(a) 黒 | 右側から出現、動いている車に触れるまで静止 | 動いている車: +車数×10点、speed設定 |
| cross(青) | line+char(e) 青 | 上方向レーン変更の指示、タップで直進に切替可能 | 車が指示に従って進路変更 |
| cross(赤) | line+char(c) 赤 | 下方向レーン変更の指示、タップで直進に切替可能 | 車が指示に従って進路変更 |
| cross(灰) | line+char(d) 灰 | 直進の指示、タップでレーン変更に切替可能 | 車がそのまま直進 |
| stop | char(f) 黒 | 停止標識、ランダムなレーンに出現 | 車が消失、得点なし |
| lane | rect(200,1) 灰 | 5本の水平レーン | - |

## 3. デザインガイド分析

| 原則 | 評価 |
|:---|:---|
| (1) シンプルさと直感性 | レーン変更の方向を色（青=上、赤=下、灰=直進）で明示。全交差点が同時に切り替わるワンボタン操作のシンプルさ。車のキャラクター表現が可愛らしい |
| (2) 視覚的フィードバック | 交差点通過時の効果音、新しい車の起動時のパワーアップ音、衝突時のパーティクル。スクロールにより速度感を表現 |
| (3) リスク・リワード | 車を増やすほど高得点を狙えるが、全車を安全に誘導する難易度が上がる。停止標識の配置を見極め、最適なレーン選択が求められる |
| (4) 斬新なメカニクス | 交差点の状態を一括変更し、複数の車が異なるタイミングで異なる交差点を通過することで生まれる戦略性。車の追加システムによる段階的な難易度上昇 |

## 4. タグ詳細

| タグ | 役割 | 戦略的意義 |
|:---|:---|:---|
| #player-multiple | 複数の車を同時に操作 | 各車の位置と次の交差点の関係を把握し、全車が安全に通過できるタイミングを見極める必要がある。一度の判断ミスが複数の車に影響 |
| #field-lanes | 5本の水平レーンで構成される道路 | レーン間の移動が戦略の核。どのレーンが安全か、どのレーンに停止標識が多いかを判断し、車を適切に配置 |
| #on_got_item-spawn | 動いている車に停止中の車が触れると、その車が起動 | 新しい車の追加タイミングを制御可能。意図的に新しい車を避けることで難易度を調整できる戦略性 |
| #on_pressed-reverse_state | タップで全交差点の状態を切り替え | 直進とレーン変更を動的に切り替えることで、刻々と変わる状況に対応。事前計画と即座の判断の両方が必要 |
| #field-auto_scroll | 最前列の車を基準に画面が自動スクロール | 先頭の車が基準となるため、車を広範囲に配置すると視界が狭まる。集団を維持するか分散させるかの判断が重要 |

## 5. 変奏ヒント

| 手法 | アイデア |
|:---|:---|
| Substitute (代用) | 車を船に変え、川の流れに沿って進む水路ゲームにする。交差点を合流点に置き換え |
| Combine (結合) | 各車に異なる色を付け、同じ色の交差点のみ反応する色マッチング要素を追加 |
| Modify (修正) | 交差点の切り替えを個別に制御できるようにし、より細かい戦略を可能にする（タップ位置で選択） |
